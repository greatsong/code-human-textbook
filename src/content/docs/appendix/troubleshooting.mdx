## Thonny 연결 문제

### Pico가 Thonny에서 인식되지 않아요

**증상:** Thonny 우측 하단에 "포트를 찾을 수 없음" 또는 연결 실패 메시지가 나와요.

**해결 방법:**
1. USB 케이블이 **데이터 전송용**인지 확인하세요. 충전 전용 케이블은 데이터를 전송할 수 없어요.
2. Thonny 메뉴에서 **Run > Configure interpreter**를 열어요.
3. 인터프리터를 **"MicroPython (Raspberry Pi Pico)"**로 선택해요.
4. 포트를 **자동 감지** 또는 수동으로 선택해요.
   - Windows: `COM3`, `COM4` 등
   - Mac: `/dev/tty.usbmodem...`
   - Linux: `/dev/ttyACM0`
5. 그래도 안 되면 USB 케이블을 뽑았다가 다시 연결하세요.

### Pico가 응답하지 않아요 (멈춤)

**증상:** Thonny에서 코드를 실행하려 하면 반응이 없어요.

**해결 방법:**
1. Thonny 상단의 **빨간 정지 버튼**을 클릭하세요.
2. 그래도 안 되면 **Ctrl+C**를 반복해서 눌러보세요.
3. 최후의 수단: USB 케이블을 뽑고, Pico의 **BOOTSEL 버튼을 누른 채** 다시 연결하세요. 그러면 Pico가 USB 드라이브로 잡혀요.
4. MicroPython 펌웨어(.uf2 파일)를 드래그 앤 드롭하면 초기화돼요.

### "Device is busy" 에러

**증상:** `Device is busy or does not respond` 메시지가 나와요.

**해결 방법:**
- `main.py`에 무한 루프 코드가 저장되어 있으면 Pico가 시작하자마자 코드를 실행하기 때문에 Thonny가 연결할 틈이 없어요.
- **Ctrl+C**를 빠르게 여러 번 눌러서 실행 중인 코드를 중단하세요.
- 그래도 안 되면 BOOTSEL 버튼으로 펌웨어를 다시 설치하세요.

## I2C 장치 인식 문제

### "OSError: [Errno 5] EIO" 에러

**증상:** I2C 센서를 읽으려 할 때 `EIO` 에러가 나요.

**해결 방법:**
1. 먼저 `i2c.scan()`으로 장치가 인식되는지 확인하세요:
   ```python
   from machine import I2C, Pin
   i2c = I2C(0, sda=Pin(8), scl=Pin(9))
   print(i2c.scan())
   ```
2. 빈 리스트 `[]`가 나오면 하드웨어 연결 문제예요:
   - Grove 케이블이 **I2C 포트**에 꽂혀 있는지 확인하세요 (ADC나 Digital이 아닌지)
   - 케이블을 뽑았다가 단단히 다시 꽂아보세요
   - 다른 I2C 포트로 바꿔보세요
3. 주소가 나오지만 읽기에서 에러가 나면:
   - 센서 초기화 후 `time.sleep(0.1)` 대기 시간을 추가하세요
   - 센서 전원을 껐다가 켜보세요 (USB 뽑았다 연결)

### I2C 주소가 예상과 달라요

**증상:** `i2c.scan()`에서 예상과 다른 주소가 나와요.

**해결 방법:**
- 일부 센서는 주소를 변경할 수 있어요. 데이터시트를 확인하세요.
- BMP280은 SDO 핀에 따라 `0x76` 또는 `0x77`이 될 수 있어요.
- 같은 주소의 장치 2개를 동시에 연결하면 충돌이 발생해요.

## 센서 값 이상

### 센서 값이 항상 0이에요

**해결 방법:**
- ADC 센서: 포트 번호를 확인하세요. A0=GP26, A1=GP27, A2=GP28이에요.
- I2C 센서: `i2c.scan()`으로 장치가 인식되는지 확인하세요.
- 케이블이 완전히 꽂혀 있는지 확인하세요.

### 센서 값이 튀어요 (노이즈)

**해결 방법:**
- **이동 평균**을 적용하세요:
  ```python
  readings = []
  while True:
      val = sensor.read_u16()
      readings.append(val)
      if len(readings) > 10:
          readings.pop(0)
      avg = sum(readings) / len(readings)
      print(f"평균: {avg}")
  ```
- ADC 센서는 원래 약간의 노이즈가 있어요. 이동 평균 10~20개면 충분히 안정적이에요.
- 센서 케이블이 너무 길거나 다른 전선과 가까이 있으면 노이즈가 심해질 수 있어요.

### 온습도 센서 값이 비현실적이에요

**해결 방법:**
- DHT20은 전원 연결 후 1~2초 대기가 필요해요. `time.sleep(0.1)` 이후에 읽으세요.
- 첫 1~2회 읽기 값은 부정확할 수 있어요. 무시하고 3번째부터 사용하세요.
- 온도가 -40도이거나 습도가 0%라면 센서 통신이 실패한 거예요.

## OLED 디스플레이 문제

### "No module named 'ssd1306'" 에러

**해결 방법:**
라이브러리를 설치해야 해요:

```python
import mip
mip.install("ssd1306")
```

Thonny에서 **Tools > Manage packages**를 열고 `ssd1306`을 검색하여 설치할 수도 있어요.

:::caution[인터넷 연결 필요]
`mip.install()`은 인터넷에서 라이브러리를 다운로드해요. Pico 2 WH가 Wi-Fi에 연결되어 있거나, Thonny를 통해 설치해야 해요.
:::

### OLED 화면에 아무것도 안 나와요

**해결 방법:**
1. `i2c.scan()`으로 OLED 주소(보통 `0x3C`)가 나오는지 확인하세요.
2. `oled.show()`를 호출했는지 확인하세요. 이 함수를 호출해야 화면에 표시돼요.
3. `oled.fill(0)`으로 화면을 지운 후 `oled.text()`와 `oled.show()`를 순서대로 호출하세요.

### OLED 글자가 겹쳐요

**해결 방법:**
- 매 프레임 시작에 `oled.fill(0)`으로 화면을 완전히 지우세요.
- `oled.text()`의 y 좌표를 8 단위로 사용하세요 (글자 높이가 8px):
  - 1줄: y=0, 2줄: y=10, 3줄: y=20, ...

## 파일 저장 문제

### "OSError: [Errno 28] ENOSPC" 에러

**증상:** 파일에 쓰려고 할 때 용량 부족 에러가 나요.

**해결 방법:**
- Pico의 저장 공간은 약 **800KB**예요. 파일이 너무 커졌어요.
- 불필요한 파일을 삭제하세요:
  ```python
  import os
  print(os.listdir())     # 파일 목록
  os.remove("old_data.csv")  # 파일 삭제
  ```
- 파일 크기 제한을 코드에 추가하세요:
  ```python
  import os
  file_size = os.stat("data.csv")[6]
  if file_size > 500000:  # 500KB
      print("파일 크기 제한!")
  ```

### main.py를 삭제하고 싶어요

**해결 방법:**
Thonny의 **Files** 패널에서 Pico 쪽의 `main.py`를 마우스 오른쪽 클릭하고 **Delete**를 선택하세요.

또는 쉘에서:
```python
import os
os.remove("main.py")
```

## Serial 통신 문제

### PC에서 Pico의 serial 데이터를 받을 수 없어요

**해결 방법:**
1. **Thonny를 완전히 종료**하세요. Thonny가 serial 포트를 점유하고 있으면 다른 프로그램이 사용할 수 없어요.
2. 코드를 `main.py`로 Pico에 저장한 뒤, Thonny를 닫고 PC Python에서 serial 포트를 여세요.
3. 포트 이름을 정확히 입력하세요:
   - Windows: 장치 관리자에서 COM 포트 번호를 확인
   - Mac: 터미널에서 `ls /dev/tty.usb*` 실행
   - Linux: `ls /dev/ttyACM*` 실행

## 그래도 해결되지 않을 때

1. **에러 메시지를 정확히 읽으세요.** 에러의 마지막 줄에 원인이 적혀 있어요.
2. **에러 메시지를 검색하세요.** 구글에 에러 메시지를 그대로 복사해서 검색하면 해결책을 찾을 수 있어요.
3. **바이브 코딩을 활용하세요.** AI에게 에러 메시지와 코드를 보여주고 해결 방법을 물어보세요.
4. **코드 레퍼런스를 참고하세요.** [코드 레퍼런스](/ai-physical-computing/appendix/code-reference/) 페이지에서 작동하는 기본 코드를 확인하세요.
5. **선생님께 질문하세요.** 시도한 것과 에러 메시지를 함께 보여주면 더 빠르게 해결할 수 있어요.
