## 준비물

| 준비물 | 설명 |
|--------|------|
| Raspberry Pi Pico WH | MicroPython이 동작하는 마이크로컨트롤러입니다. WH 모델은 헤더 핀이 미리 납땜되어 있어 바로 사용할 수 있습니다. |
| Grove Shield for Pi Pico | Pico 위에 올려 꽂는 확장 보드입니다. Grove 커넥터 포트가 여러 개 있어 센서를 케이블 하나로 연결할 수 있습니다. |
| Grove 케이블 (4핀) | 센서와 Shield를 연결하는 케이블입니다. 양쪽 끝에 4핀 커넥터가 달려 있습니다. |
| 온습도 센서 (DHT20) | 공기 중의 온도와 습도를 동시에 측정하는 디지털 센서입니다. I2C 방식으로 통신합니다. ([센서 도감에서 자세히 보기](/code-human-textbook/appendix/sensor-catalog/)) |
| USB-C 케이블 | Pico와 컴퓨터를 연결하여 전원을 공급하고 코드를 전송합니다. |

### 연결 방법

DHT20 센서를 Grove 케이블로 Shield의 **I2C 포트**(I2C0 또는 I2C1)에 연결하세요.

:::tip[Grove 커넥터 안내]
4핀 커넥터라 방향이 정해져 있어서, 홈에 맞춰 끝까지 밀어 넣으면 됩니다. 잘못 꽂을 걱정이 없습니다.
:::

:::note[포트 찾기]
Grove Shield에는 여러 종류의 포트가 있습니다. DHT20은 반드시 **"I2C"라고 적힌 포트**(I2C0 또는 I2C1)에 연결해야 합니다. 아날로그(A0~A2)나 디지털(D16~D20) 포트에 꽂으면 동작하지 않습니다.
:::

## 활동 가이드

### Step 1. 연결하기 (5분)

1. **Pico WH를 Grove Shield 위에 올려 꽂습니다.** Pico의 핀과 Shield의 소켓을 맞춰 끝까지 눌러주세요.
2. **USB-C 케이블로 컴퓨터와 연결합니다.** Pico의 USB-C 포트에 케이블을 꽂으면 전원이 들어옵니다.
3. **DHT20 센서를 Grove 케이블로 I2C 포트에 꽂습니다.** Shield에 "I2C0" 또는 "I2C1"이라고 적힌 포트를 찾아 연결하세요.

:::tip
커넥터 양쪽의 홈을 확인하세요. 한 방향으로만 들어가도록 설계되어 있습니다.
:::

### Step 2. 첫 코드 작성 (15분)

1. 컴퓨터에서 **Thonny**를 열어주세요. (설치가 아직이라면 [시작 가이드](/code-human-textbook/guide/setup/)를 먼저 확인하세요.)
2. 아래 [기본 코드](#기본-코드)를 Thonny 편집기에 입력합니다.
3. 상단의 **실행(Run)** 버튼을 클릭합니다.
4. 하단 **Shell** 창에 온도와 습도가 2초마다 출력되는지 확인하세요.

:::note[생각해보기]
손으로 센서를 감싸면 온도가 올라가나요? 올라간다면 얼마나 걸리나요? 손을 떼면 원래 온도로 돌아오는 데 얼마나 걸리나요?
:::

### Step 3. 교실 온도 지도 만들기 (20분)

교실 안 **4~5곳**의 위치에서 온도와 습도를 측정해봅시다. 노트북과 Pico를 함께 들고 이동하면서, 각 위치에서 값이 안정될 때까지(약 30초) 기다린 뒤 기록합니다.

:::tip
노트북이 무거우면, 짝과 역할을 나눠보세요. 한 사람은 노트북을 들고, 한 사람은 센서를 위치에 대고 측정합니다.
:::

**측정 위치 예시:** 창가, 복도 쪽, 에어컨 앞, 교실 중앙, 문 옆

|-----------|--------------------------|---------------|--------------|------|

| 에어컨 앞 | | | | |

### Step 4. 토론하기 (10분)

다음 질문을 모둠에서 이야기해보세요.

1. **"가장 온도 차이가 큰 두 곳은 어디인가요?"** 그 차이가 생기는 이유는 무엇일까요?
2. **"체감 온도와 센서 측정값이 차이나는 곳이 있나요?"** 왜 그럴까요? (힌트: 바람, 습도, 직전 활동 등)
3. **"이 데이터로 교실 환경을 개선할 수 있는 방법은?"** 자리 배치, 환기 시점, 에어컨 방향 등 구체적으로 제안해봅시다.

### 기본 코드

```python
from machine import I2C, Pin
import time

# I2C 통신 설정 (Grove Shield I2C0 포트)
i2c = I2C(0, sda=Pin(8), scl=Pin(9))

# DHT20 센서 초기화
sensor_addr = 0x38

time.sleep(0.1)  # 센서 초기화 대기

def read_dht20():
    """DHT20 센서에서 온도와 습도를 읽습니다"""
    i2c.writeto(sensor_addr, bytes([0xAC, 0x33, 0x00]))
    time.sleep(0.08)  # 측정 대기
    data = i2c.readfrom(sensor_addr, 7)

    humidity = ((data[1] << 12) | (data[2] << 4) | (data[3] >> 4)) / 1048576 * 100
    temperature = (((data[3] & 0x0F) << 16) | (data[4] << 8) | data[5]) / 1048576 * 200 - 50

    return round(temperature, 1), round(humidity, 1)

# 2초마다 측정하여 출력
while True:
    temp, humi = read_dht20()
    print(f"온도: {temp}°C, 습도: {humi}%")
    time.sleep(2)
```

### 줄별 해설

| 줄 | 코드 | 설명 |
|----|------|------|
| 5 | `I2C(0, sda=Pin(8), scl=Pin(9))` | Grove Shield의 I2C0 포트를 사용합니다. I2C는 센서와 마이크로컨트롤러가 데이터를 주고받는 통신 방식입니다. |
| 8 | `0x38` | DHT20 센서의 I2C 주소입니다. 센서마다 고유한 주소가 있어, 여러 센서를 하나의 포트에 연결해도 구분할 수 있습니다. |
| 13 | `i2c.writeto(...)` | 센서에 "측정을 시작하라"는 명령을 보냅니다. `0xAC, 0x33, 0x00`은 DHT20이 이해하는 명령어입니다. |
| 17~18 | `humidity = ...` / `temperature = ...` | 센서가 보내준 원시 데이터(비트 단위)를 사람이 이해할 수 있는 온도(°C)와 습도(%) 숫자로 변환합니다. |
| 10 | `time.sleep(0.1)` | 센서가 전원을 받고 준비되는 시간(100ms)을 기다립니다. 이 줄이 없으면 첫 번째 측정이 실패할 수 있습니다. |
| 24 | `print(f"온도: {temp}°C ...")` | f-string은 문자열 안에 변수 값을 넣는 Python 문법입니다. 중괄호 `{}` 안에 변수 이름을 쓰면 됩니다. |

### 도전 코드: 쾌적도 판정

기본 코드의 `read_dht20()` 함수는 그대로 사용하고, `while True:` 부분만 아래로 바꿔보세요.

```python
# 쾌적도 판정 추가
while True:
    temp, humi = read_dht20()

    if 20 <= temp <= 26 and 40 <= humi <= 60:
        status = "쾌적"
    elif temp > 28 or humi > 70:
        status = "불쾌"
    else:
        status = "보통"

    print(f"온도: {temp}°C, 습도: {humi}% → {status}")
    time.sleep(2)
```

:::note[코드 읽어보기]
`if` 문은 조건에 따라 다른 동작을 하도록 하는 문법입니다. `20 <= temp <= 26`은 "온도가 20도 이상이고 26도 이하"라는 뜻입니다. 온도와 습도 조건을 여러분의 기준에 맞게 바꿔보세요.
:::

## 잠깐, 방금 무슨 일이 일어난 걸까요?

여러분이 방금 한 일을 다시 생각해보세요.

공기 중의 수증기와 열에너지가
**→** DHT20 센서 내부의 감지 소자에서 **전기 신호**로 바뀌고
**→** I2C 통신을 통해 **디지털 데이터**로 전송되고
**→** 코드가 그 데이터를 사람이 읽을 수 있는 **숫자**로 변환했습니다.

스마트폰의 날씨 앱, 에어컨의 자동 온도 조절, 기상청의 기상 관측 — 모두 이 원리의 확장입니다. 여러분은 방금 그 원리를 직접 구현한 것입니다.

범위가 크다면 교실 안에서도 위치에 따라 온도가 많이 다르다는 뜻입니다. 범위가 1°C 이내라면 교실 전체가 비교적 균일한 온도라고 할 수 있습니다. 여러분의 교실은 어떤가요?
