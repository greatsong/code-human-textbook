## 준비물

| 준비물 | 설명 |
|--------|------|
| Raspberry Pi Pico 2 WH | 마이크로컨트롤러 |
| Grove Shield for Pi Pico | 확장 보드 |
| 온습도 센서 (DHT20) | I2C 포트에 연결 |
| 빛 센서 (Light Sensor v1.2) | A0 포트에 연결 |
| OLED 디스플레이 (SSD1306) | I2C 포트에 연결 — **DHT20과 같은 I2C 버스 공유 가능** |
| Grove 케이블 3개 | 센서 2개 + 디스플레이 1개 |

### 연결 방법

- **DHT20** → I2C1 포트
- **OLED** → I2C1 포트 (DHT20과 함께 **I2C Hub**를 통해 연결)
- **빛 센서** → A0 포트

:::tip[I2C 버스 공유]
DHT20(주소 0x38)과 OLED(주소 0x3C)는 둘 다 I2C 통신을 사용하지만 주소가 다릅니다. 코드에서 `I2C(1)`으로 같은 버스를 공유하므로, **두 장치를 같은 I2C1 버스에 연결**해야 합니다. Grove Shield의 I2C1 포트가 1개뿐이므로, **I2C Hub**를 사용하여 여러 장치를 연결하세요.
:::

## 활동 가이드

### Step 1. 세 장치 연결하기 (5분)

1. I2C Hub를 I2C1 포트에 연결하고, DHT20과 OLED를 Hub에 꽂습니다.
2. 빛 센서를 A0 포트에 연결합니다.
3. USB-C로 컴퓨터와 연결합니다.

:::note[확인하기]
Thonny Shell에서 I2C 장치를 확인할 수 있습니다:
```python
from machine import I2C, Pin
i2c = I2C(1, sda=Pin(6), scl=Pin(7))
print([hex(a) for a in i2c.scan()])
# ['0x38', '0x3c'] → DHT20과 OLED가 보입니다
```
:::

### Step 2. OLED에 글자 띄우기 (10분)

1. 아래 [기본 코드](#기본-코드)의 OLED 부분을 먼저 실행합니다.
2. OLED 화면에 "Hello!"가 나타나는지 확인하세요.
3. 텍스트를 바꿔보세요 — 한글은 지원되지 않으니 영어와 숫자를 사용합니다.

### Step 3. 실시간 대시보드 만들기 (25분)

1. [기본 코드](#기본-코드) 전체를 실행합니다.
2. OLED에 온도, 습도, 밝기가 실시간으로 업데이트되는지 확인합니다.
3. 센서를 손으로 감싸거나, 빛을 가려보면서 값이 변하는 걸 관찰합니다.

:::tip[대시보드 레이아웃]
OLED 해상도는 128x64 픽셀입니다. `oled.text()`는 8x8 픽셀 글자를 표시합니다. 한 줄에 16글자, 총 8줄까지 쓸 수 있습니다. 어떤 정보를 어디에 배치할지 종이에 먼저 그려보세요.
:::

### Step 4. 나만의 대시보드 꾸미기 (10분)

1. [도전 코드](#도전-코드)를 참고하여 레이아웃을 바꿔봅니다.
2. 구분선, 제목, 아이콘(기호)을 추가해봅니다.
3. 모둠과 대시보드 디자인을 비교해봅시다.

### 기본 코드

```python
from machine import I2C, Pin, ADC
from ssd1306 import SSD1306_I2C
import time

# I2C 설정
i2c = I2C(1, sda=Pin(6), scl=Pin(7))

# OLED 디스플레이 초기화 (128x64)
oled = SSD1306_I2C(128, 64, i2c)

# DHT20 센서
sensor_addr = 0x38
time.sleep(0.1)

# 빛 센서 (A0)
light = ADC(Pin(26))

def read_dht20():
    i2c.writeto(sensor_addr, bytes([0xAC, 0x33, 0x00]))
    time.sleep(0.08)
    data = i2c.readfrom(sensor_addr, 7)
    humidity = ((data[1] << 12) | (data[2] << 4) | (data[3] >> 4)) / 1048576 * 100
    temperature = (((data[3] & 0x0F) << 16) | (data[4] << 8) | data[5]) / 1048576 * 200 - 50
    return round(temperature, 1), round(humidity, 1)

# 대시보드 루프
while True:
    temp, humi = read_dht20()
    bright = round(light.read_u16() / 65535 * 100, 1)

    oled.fill(0)  # 화면 지우기
    oled.text("=== Dashboard ===", 0, 0)
    oled.text(f"Temp:  {temp} C", 0, 16)
    oled.text(f"Humi:  {humi} %", 0, 28)
    oled.text(f"Light: {bright} %", 0, 40)
    oled.show()  # 화면 전송

    time.sleep(1)
```

### 줄별 해설

| 줄 | 코드 | 설명 |
|----|------|------|
| 7 | `SSD1306_I2C(128, 64, i2c)` | 128x64 픽셀 OLED를 I2C로 초기화합니다. `ssd1306` 라이브러리가 필요합니다. |
| 27 | `oled.fill(0)` | 화면을 검은색(0)으로 채워 이전 내용을 지웁니다. 매 프레임마다 호출합니다. |
| 28 | `oled.text("...", 0, 0)` | (x좌표, y좌표) 위치에 텍스트를 버퍼에 그립니다. 아직 화면에는 안 보입니다. |
| 32 | `oled.show()` | 버퍼에 그린 내용을 실제 OLED 화면으로 전송합니다. 이 줄이 있어야 보입니다. |

:::note[ssd1306 라이브러리 설치]
Thonny에서 **도구 → 패키지 관리**를 열고, `ssd1306`을 검색하여 설치하세요.
:::

### 도전 코드: 그래픽 대시보드

```python
from machine import I2C, Pin, ADC
from ssd1306 import SSD1306_I2C
import time

i2c = I2C(1, sda=Pin(6), scl=Pin(7))
oled = SSD1306_I2C(128, 64, i2c)
sensor_addr = 0x38
light = ADC(Pin(26))
time.sleep(0.1)

def read_dht20():
    i2c.writeto(sensor_addr, bytes([0xAC, 0x33, 0x00]))
    time.sleep(0.08)
    data = i2c.readfrom(sensor_addr, 7)
    humidity = ((data[1] << 12) | (data[2] << 4) | (data[3] >> 4)) / 1048576 * 100
    temperature = (((data[3] & 0x0F) << 16) | (data[4] << 8) | data[5]) / 1048576 * 200 - 50
    return round(temperature, 1), round(humidity, 1)

# 온도 히스토리 (바 그래프용)
temp_history = []

while True:
    temp, humi = read_dht20()
    bright = round(light.read_u16() / 65535 * 100, 1)

    # 히스토리 업데이트 (최근 32개)
    temp_history.append(temp)
    if len(temp_history) > 32:
        temp_history.pop(0)

    oled.fill(0)

    # 상단: 제목
    oled.text("AI PhysComp", 0, 0)
    oled.hline(0, 10, 128, 1)

    # 중단: 센서값
    oled.text(f"T:{temp}C", 0, 14)
    oled.text(f"H:{humi}%", 68, 14)
    oled.text(f"L:{bright}%", 0, 26)

    # 쾌적도 판정
    if 20 <= temp <= 26 and 40 <= humi <= 60:
        oled.text("[Good]", 76, 26)
    else:
        oled.text("[Alert]", 72, 26)

    # 하단: 온도 미니 바 그래프
    oled.hline(0, 38, 128, 1)
    oled.text("Temp Trend", 0, 40)
    for i, t in enumerate(temp_history):
        bar_h = max(1, min(12, int((t - 15) / 20 * 12)))
        oled.vline(i * 4, 64 - bar_h, bar_h, 1)

    oled.show()
    time.sleep(2)
```

:::note[대시보드 3영역 구성]
- **상단**: 제목 + 구분선
- **중단**: 센서 실시간 값 + 쾌적도 판정
- **하단**: 온도 변화 미니 바 그래프

`oled.hline(x, y, width, color)`는 가로선, `oled.vline(x, y, height, color)`는 세로선을 그립니다.
:::

## 잠깐, 방금 무슨 일이 일어난 걸까요?

여러분은 방금 **대시보드**를 만들었습니다.

센서 3개의 데이터가
**→** 코드로 읽히고
**→** 하나의 화면에 정리되고
**→** 실시간으로 업데이트됩니다.

자동차의 계기판, 비행기의 콕핏, 공장의 모니터링 시스템 — 모두 이런 대시보드입니다. 여러분은 방금 그 원리를 128x64 픽셀 안에 구현한 것입니다.

다음 막에서는 이런 대시보드를 **직접 코딩하지 않고, AI에게 시켜서** 만들어봅니다.

## 대시보드 디자인 챌린지

OLED 대시보드 레이아웃을 자유롭게 설계해보세요.

1. 128x64 픽셀 격자에 어떤 정보를 어디에 배치할지 종이에 먼저 그립니다.
2. 다음 질문에 답하며 설계합니다:
   - 가장 중요한 정보는 무엇인가요? → 크게, 상단에 배치
   - 경고가 필요한 상황은 언제인가요? → 깜빡임이나 반전 표시
   - 시간에 따른 변화를 보여줘야 하나요? → 그래프 영역 확보

| 대시보드 유형 | 설명 | 적합한 상황 |
|--------------|------|------------|
| 숫자형 | 큰 글자로 핵심 값만 표시 | 멀리서도 봐야 할 때 |
| 그래프형 | 시간에 따른 변화 추이 | 패턴을 찾고 싶을 때 |
| 상태형 | Good/Alert 등 판정 결과 | 빠른 판단이 필요할 때 |
| 복합형 | 숫자 + 그래프 + 상태 | 종합 모니터링 |
