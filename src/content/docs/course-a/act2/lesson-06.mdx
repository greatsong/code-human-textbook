### 연결 방법

세 장치 모두 **I2C 통신**을 사용합니다. Grove 케이블로 Shield의 **I2C 포트**에 각각 연결하세요.

:::tip[I2C 주소가 다르면 같은 버스에 연결 가능]
SCD41(주소 0x62), DHT20(주소 0x38), OLED SSD1306(주소 0x3C)은 모두 I2C 주소가 다릅니다. 그래서 같은 I2C1 버스(SDA=GP6, SCL=GP7)에 동시에 연결해도 충돌 없이 동작합니다.
:::

:::note[SCD41 센서 주의사항]
SCD41은 측정을 시작한 뒤 **약 5초**가 지나야 데이터가 준비됩니다. 처음 실행할 때 잠시 기다려야 하는 이유입니다.
:::

### 보드 포트 맵

세 장치 모두 I2C 통신을 사용합니다. 주소가 다르므로 같은 I2C 버스에 연결해도 충돌하지 않습니다.

:::note[I2C 포트가 2개뿐?]
Grove Shield에는 I2C 포트가 2개(I2C0, I2C1)뿐입니다. 3개 이상의 I2C 장치를 사용할 때는 **I2C Hub**를 I2C1 포트에 연결하면 최대 5개 장치를 동시에 사용할 수 있습니다. 주소가 다른 I2C 장치는 같은 버스를 공유해도 충돌하지 않습니다.
:::

### AI에게 보낼 프롬프트

아래 프롬프트를 AI 챗봇에 복사해서 보내보세요. 필요에 따라 수정해도 좋습니다.

```
Pico 2 WH에 Grove SCD41 CO2 센서와 DHT20 온습도 센서를 I2C1(SDA=GP6, SCL=GP7)에,
OLED SSD1306도 같은 I2C에 연결했어.

CO2 농도, 온도, 습도를 OLED 대시보드에 실시간 표시하고,
CO2가 1000ppm 넘으면 'VENTILATE!' 경고를 표시하는 MicroPython 코드 만들어줘.

조건:
- SCD41 I2C 주소는 0x62
- 측정 시작 명령: bytes([0x21, 0xB1])
- 5초 대기 후 읽기 명령: bytes([0xEC, 0x05])
- 9바이트 읽기
- CO2 = (data[0]<<8)|data[1]
- 온도 = -45 + 175 * ((data[3]<<8)|data[4]) / 65535
- 습도 = 100 * ((data[6]<<8)|data[7]) / 65535
- DHT20 주소 0x38
- OLED 주소 0x3C, ssd1306 라이브러리 사용
- 대시보드 레이아웃: 상단에 CO2 큰 숫자, 중단에 온도/습도, 하단에 상태(좋음/보통/환기필요)
```

:::tip[프롬프트가 길수록 정확해요]
하드웨어 정보, 센서 통신 프로토콜, 원하는 레이아웃까지 구체적으로 알려주면 AI가 더 정확한 코드를 생성합니다. 5차시에서 배운 단계적 요청도 효과적입니다.
:::

### 기본 코드

AI가 생성할 수 있는 완성 코드 예시입니다. AI 응답이 작동하지 않을 때 참고하세요.

```python
from machine import I2C, Pin
import ssd1306
import time

# I2C1 설정 (SCD41, DHT20, OLED 모두 같은 버스)
i2c = I2C(1, sda=Pin(6), scl=Pin(7))

# OLED 설정
oled = ssd1306.SSD1306_I2C(128, 64, i2c, addr=0x3C)

# SCD41 주소
SCD41_ADDR = 0x62

# DHT20 주소
DHT20_ADDR = 0x38

def read_scd41():
    """SCD41에서 CO2, 온도, 습도를 읽어 반환합니다"""
    # 측정 시작 명령
    i2c.writeto(SCD41_ADDR, bytes([0x21, 0xB1]))
    time.sleep(5)  # 측정 대기

    # 데이터 읽기 명령
    i2c.writeto(SCD41_ADDR, bytes([0xEC, 0x05]))
    time.sleep(0.01)
    data = i2c.readfrom(SCD41_ADDR, 9)

    co2 = (data[0] << 8) | data[1]
    temp = -45 + 175 * ((data[3] << 8) | data[4]) / 65535
    humi = 100 * ((data[6] << 8) | data[7]) / 65535

    return co2, round(temp, 1), round(humi, 1)

def read_dht20():
    """DHT20에서 온도와 습도를 읽어 반환합니다"""
    i2c.writeto(DHT20_ADDR, bytes([0xAC, 0x33, 0x00]))
    time.sleep(0.08)
    data = i2c.readfrom(DHT20_ADDR, 7)

    humidity = ((data[1] << 12) | (data[2] << 4) | (data[3] >> 4)) / 1048576 * 100
    temperature = (((data[3] & 0x0F) << 16) | (data[4] << 8) | data[5]) / 1048576 * 200 - 50

    return round(temperature, 1), round(humidity, 1)

def get_status(co2):
    """CO2 수치에 따라 상태 문자열을 반환합니다"""
    if co2 < 600:
        return "GOOD"
    elif co2 < 1000:
        return "NORMAL"
    else:
        return "VENTILATE!"

def show_dashboard(co2, temp, humi, status):
    """OLED에 대시보드를 표시합니다"""
    oled.fill(0)  # 화면 지우기

    # 상단: CO2 큰 숫자
    oled.text("CO2", 0, 0)
    oled.text(f"{co2} ppm", 30, 0)

    # 구분선
    oled.hline(0, 14, 128, 1)

    # 중단: 온도와 습도
    oled.text(f"Temp: {temp} C", 0, 20)
    oled.text(f"Humi: {humi} %", 0, 32)

    # 구분선
    oled.hline(0, 44, 128, 1)

    # 하단: 상태
    oled.text(f"[{status}]", 0, 50)

    oled.show()

# 메인 루프
print("환경 모니터링 대시보드 시작...")
print("SCD41 첫 측정까지 약 5초 소요")

count = 0
while True:
    count += 1

    # SCD41에서 CO2 + 온습도 읽기
    co2, temp_scd, humi_scd = read_scd41()

    # DHT20에서 온습도 읽기 (비교용)
    temp_dht, humi_dht = read_dht20()

    # 상태 판정
    status = get_status(co2)

    # OLED 대시보드 표시
    show_dashboard(co2, temp_scd, humi_scd, status)

    # Shell에도 출력 (기록용)
    print(f"[{count}] CO2:{co2}ppm T:{temp_scd}C H:{humi_scd}% | DHT20 T:{temp_dht}C H:{humi_dht}% | {status}")

    time.sleep(5)  # SCD41은 최소 5초 간격
```

### 줄별 해설

| 줄 | 코드 | 설명 |
|----|------|------|
| 6 | `I2C(1, sda=Pin(6), scl=Pin(7))` | I2C1 버스를 설정합니다. SCD41, DHT20, OLED가 모두 이 버스를 공유합니다. |
| 18 | `bytes([0x21, 0xB1])` | SCD41에 "측정을 시작하라"는 명령을 보냅니다. |
| 19 | `time.sleep(5)` | SCD41이 CO2를 측정하는 데 약 5초가 걸립니다. |
| 21 | `bytes([0xEC, 0x05])` | "측정 결과를 보내라"는 읽기 명령입니다. |
| 24-26 | CO2, temp, humi 계산 | 9바이트의 원시 데이터를 사람이 이해할 수 있는 단위(ppm, °C, %)로 변환합니다. |
| 43-45 | `get_status()` | CO2 600ppm 미만은 좋음, 1000ppm 미만은 보통, 이상은 환기 필요입니다. |

### 도전 코드: CO2 바 그래프 + 히스토리

OLED에 CO2 수치를 바 그래프로 표시하고, 최근 측정값의 추이를 그래프로 보여줍니다.

```python
from machine import I2C, Pin
import ssd1306
import time

i2c = I2C(1, sda=Pin(6), scl=Pin(7))
oled = ssd1306.SSD1306_I2C(128, 64, i2c, addr=0x3C)
SCD41_ADDR = 0x62

def read_scd41():
    i2c.writeto(SCD41_ADDR, bytes([0x21, 0xB1]))
    time.sleep(5)
    i2c.writeto(SCD41_ADDR, bytes([0xEC, 0x05]))
    time.sleep(0.01)
    data = i2c.readfrom(SCD41_ADDR, 9)
    co2 = (data[0] << 8) | data[1]
    temp = -45 + 175 * ((data[3] << 8) | data[4]) / 65535
    humi = 100 * ((data[6] << 8) | data[7]) / 65535
    return co2, round(temp, 1), round(humi, 1)

def get_status(co2):
    if co2 < 600:
        return "GOOD"
    elif co2 < 1000:
        return "OK"
    else:
        return "VENT!"

# CO2 히스토리 (최근 20개)
history = []

def show_dashboard_with_graph(co2, temp, humi, status):
    oled.fill(0)

    # 상단: CO2 수치 + 상태
    oled.text(f"CO2:{co2}ppm", 0, 0)
    oled.text(f"[{status}]", 88, 0)

    # 중단: 온도/습도
    oled.text(f"T:{temp}C H:{humi}%", 0, 12)

    # CO2 바 그래프 (0~2000ppm을 0~128픽셀로)
    bar_width = min(128, int(co2 / 2000 * 128))
    oled.fill_rect(0, 24, bar_width, 6, 1)
    oled.rect(0, 24, 128, 6, 1)  # 테두리

    # 1000ppm 기준선 표시
    line_x = int(1000 / 2000 * 128)
    oled.vline(line_x, 22, 10, 1)

    # 하단: 히스토리 그래프 (최근 20개)
    if len(history) >= 2:
        graph_y = 34
        graph_h = 28
        for i in range(1, len(history)):
            # CO2를 그래프 높이로 변환 (400~1500ppm 범위)
            y1 = graph_y + graph_h - int((history[i-1] - 400) / 1100 * graph_h)
            y2 = graph_y + graph_h - int((history[i] - 400) / 1100 * graph_h)
            y1 = max(graph_y, min(graph_y + graph_h, y1))
            y2 = max(graph_y, min(graph_y + graph_h, y2))
            x1 = (i - 1) * (128 // 20)
            x2 = i * (128 // 20)
            oled.line(x1, y1, x2, y2, 1)

    oled.show()

print("CO2 히스토리 대시보드 시작...")

while True:
    co2, temp, humi = read_scd41()
    status = get_status(co2)

    # 히스토리 업데이트 (최대 20개 유지)
    history.append(co2)
    if len(history) > 20:
        history.pop(0)

    show_dashboard_with_graph(co2, temp, humi, status)
    print(f"CO2:{co2}ppm T:{temp}C H:{humi}% [{status}] history:{len(history)}")

    time.sleep(5)
```

:::note[코드 읽어보기]
`history` 리스트에 최근 20개의 CO2 측정값을 저장합니다. 새 값이 추가되면 가장 오래된 값을 `pop(0)`으로 제거합니다. OLED 하단에는 이 히스토리를 꺾은선 그래프로 그려서, CO2가 올라가고 있는지 내려가고 있는지 한눈에 볼 수 있습니다. `oled.line(x1, y1, x2, y2, 1)`은 두 점을 잇는 직선을 그리는 함수입니다.
:::

## 잠깐, 방금 무슨 일이 일어난 걸까요?

여러분이 방금 한 일을 다시 생각해보세요.

AI에게 **프롬프트로 코드를 요청**하고
**→** AI가 **3개 센서를 제어하는 코드를 생성**하고
**→** 여러분이 코드를 **리뷰하고 수정**하고
**→** Pico가 CO2, 온도, 습도를 **동시에 측정**하고
**→** OLED에 **실시간 대시보드**가 표시되었습니다.

1막에서는 센서 하나를 읽는 코드를 직접 작성했습니다. 2막에서는 바이브 코딩으로 **센서 3개 + OLED를 동시에 제어하는 시스템**을 만들었습니다. 직접 코딩한다면 한 시간 이상 걸릴 코드를, AI의 도움으로 빠르게 완성한 것입니다.

하지만 이것이 가능했던 이유는, 여러분이 1막에서 **I2C가 무엇인지**, **센서 데이터를 어떻게 읽는지** 이미 알고 있었기 때문입니다.

{/* 섹션 F: 대시보드/데이터 활동 */}

## 대시보드 데이터로 생각하기

환기 전후 기록표의 데이터를 분석해봅시다.

1. **환기 전 CO2 최고치**는 얼마인가요?
2. **환기 시작 후 CO2가 600ppm 이하로 떨어지는 데 몇 분** 걸렸나요?
3. **온도는 환기 전후로 얼마나 변했나요?** CO2 감소와 온도 변화는 트레이드오프 관계인가요?

#### 교실 환경 기준 비교

| CO2 농도 | 상태 | 비고 |
|---------|------|------|
| 400ppm 이하 | 실외 수준 | 맑은 날 야외 공기 |
| 400~600ppm | 매우 좋음 | 환기가 잘 된 교실 |
| 600~1000ppm | 보통 | 일반적인 교실 |
| 1000~1500ppm | 환기 필요 | 집중력 저하 시작 |
| 1500ppm 이상 | 즉시 환기 | 졸음, 두통 가능 |

:::note[CO2와 집중력]
연구에 따르면 CO2 농도가 1000ppm을 넘으면 학생들의 집중력과 판단력이 떨어지기 시작합니다. 교실에 대시보드가 있으면 "느낌"이 아니라 "데이터"로 환기 시점을 판단할 수 있습니다.
:::

## 마무리

### 오늘 배운 것

- **바이브 코딩 실전 활용**: AI에게 복잡한 멀티센서 코드를 요청하고, 리뷰하고, 수정하는 과정을 경험했습니다.
- **I2C 버스 공유**: 여러 장치가 같은 I2C 버스를 공유하되, 주소로 구분된다는 것을 확인했습니다.
- **환경 데이터 해석**: CO2, 온도, 습도의 관계를 이해하고, 환기의 효과를 데이터로 확인했습니다.
