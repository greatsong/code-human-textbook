## 준비물

| 준비물 | 설명 |
|--------|------|
| Raspberry Pi Pico 2 WH + Grove Shield | Pico 위에 Shield를 올려 꽂아 사용합니다. |
| Grove 케이블 (4핀) | 센서와 Shield를 연결하는 케이블입니다. |
| OLED 디스플레이 (SSD1306) | 128×64 픽셀의 작은 화면입니다. I2C 통신(주소 0x3C)을 사용합니다. |
| 센서 1개 (아래 중 택1) | DHT20 온습도, 소리 센서, 빛 센서 |
| I2C Y-스플리터 케이블 | DHT20 + OLED처럼 **I2C 장치 2개**를 같은 버스에 연결할 때 필요합니다. 1개의 I2C 포트를 2개로 분배합니다. |
| USB-C 케이블 | Pico와 컴퓨터를 연결합니다. |

### 연결 방법

:::tip[I2C 센서와 OLED]
DHT20(주소 0x38)과 OLED(주소 0x3C)은 모두 I2C 통신을 사용합니다. 주소가 다르므로 같은 I2C1 버스(SDA=GP6, SCL=GP7)에 연결해도 충돌 없이 동작합니다.
:::

:::note[ADC 센서 포트]
빛 센서는 **A0 포트(GP26)**, 소리 센서는 **A1 포트(GP27)**에 연결합니다. ADC 센서는 반드시 아날로그 포트(A0, A1)에 연결해야 합니다.
:::

### 프로젝트 1: 교실 온도 대시보드

**센서:** DHT20 온습도 센서 (I2C — 주소 0x38)

**아이디어:** 교실 온도와 습도를 실시간으로 측정하고, "Cold / Good / Hot" 상태를 OLED에 표시합니다. 온도 막대그래프로 한눈에 현재 상태를 파악할 수 있습니다.

**OLED 대시보드 설계:**
```
+---------------------------+
|████ Classroom ████████████|  ← 반전 헤더
|─────────────────────────--|
| 24.5 °C        H:55.2%   |
|                 [Good]    |
|─────────────────────────--|
|████ 0 ──── sparkline ─ 40C|  ← 스파크라인 영역
+---------------------------+
```

**연결:** DHT20 → I2C1 포트, OLED → 같은 I2C1 포트 (주소가 다르므로 공존 가능)

#### 전체 코드

```python
from machine import I2C, Pin
import time

# ── I2C 설정 ──
i2c = I2C(1, sda=Pin(6), scl=Pin(7), freq=100000)
sensor_addr = 0x38
time.sleep(0.1)

# ── OLED 설정 ──
from ssd1306 import SSD1306_I2C
oled = SSD1306_I2C(128, 64, i2c)

def read_dht20():
    i2c.writeto(sensor_addr, bytes([0xAC, 0x33, 0x00]))
    time.sleep(0.08)
    data = i2c.readfrom(sensor_addr, 7)
    humidity = ((data[1] << 12) | (data[2] << 4) | (data[3] >> 4)) / 1048576 * 100
    temperature = (((data[3] & 0x0F) << 16) | (data[4] << 8) | data[5]) / 1048576 * 200 - 50
    return round(temperature, 1), round(humidity, 1)

def get_status(temp):
    if temp < 18:
        return "Cold"
    elif temp > 28:
        return "Hot"
    else:
        return "Good"

# ── CSV 파일 준비 ──
import os
try:
    size = os.stat("temp_log.csv")[6]
except:
    size = 0
f = open("temp_log.csv", "a")
if size == 0:
    f.write("time_s,temp,humi\n")
start = time.ticks_ms()

# 온도 이력 (스파크라인용, 최대 50개)
temp_history = []

# ── 메인 루프 ──
while True:
    temp, humi = read_dht20()
    status = get_status(temp)
    elapsed = time.ticks_diff(time.ticks_ms(), start) // 1000

    # 이력 저장
    temp_history.append(int(temp))
    if len(temp_history) > 50:
        temp_history.pop(0)

    # OLED 대시보드
    oled.fill(0)
    # 헤더 바
    oled.fill_rect(0, 0, 128, 12, 1)
    oled.text("Classroom", 28, 2, 0)
    oled.hline(0, 13, 128, 1)
    # 온습도 표시
    oled.text(f"{temp}", 4, 16)
    oled.text("o", 4 + len(str(temp)) * 8, 14)
    oled.text("C", 4 + len(str(temp)) * 8 + 8, 16)
    oled.text(f"H:{humi:.0f}%", 72, 16)
    oled.text(f"[{status}]", 76, 28)
    # 스파크라인 영역
    oled.hline(0, 44, 128, 1)
    oled.fill_rect(0, 46, 128, 18, 1)
    oled.text("0", 0, 46, 0)
    oled.text("40C", 100, 46, 0)
    for i in range(len(temp_history)):
        h = int((temp_history[i] - 10) / 30 * 14)
        h = max(1, min(14, h))
        oled.vline(2 + i, 62 - h, h, 0)
    oled.show()

    # CSV 저장
    f.write(f"{elapsed},{temp},{humi}\n")
    f.flush()

    print(f"{elapsed}s | {temp}°C {humi}% [{status}]")
    time.sleep(2)
```

#### 줄별 해설

| 줄 | 코드 | 설명 |
|----|------|------|
| 4 | `I2C(1, sda=Pin(6), scl=Pin(7))` | Grove Shield의 I2C1 포트를 사용합니다. SDA=GP6, SCL=GP7입니다. |
| 5 | `0x38` | DHT20의 I2C 주소입니다. 모든 DHT20은 이 주소를 사용합니다. |
| 12 | `bytes([0xAC, 0x33, 0x00])` | "측정을 시작하라"는 명령입니다. DHT20이 이해하는 프로토콜입니다. |
| 15-16 | `humidity = ...` / `temperature = ...` | 7바이트 원시 데이터를 사람이 읽을 수 있는 온도(°C)와 습도(%)로 변환합니다. |
| 19-24 | `get_status()` | 18도 미만이면 "Cold", 28도 초과면 "Hot", 그 사이면 "Good"으로 판정합니다. |
| 35 | `temp_history = []` | 최근 50개의 온도 값을 저장하는 리스트입니다. OLED 스파크라인 그래프에 사용합니다. |
| 44 | `oled.fill_rect(0, 0, 128, 12, 1)` | 반전 헤더 바를 그립니다. 흰 배경 위에 검은 글씨로 제목을 표시합니다. |
| 58-61 | 스파크라인 영역 | 온도 이력을 세로선으로 그립니다. 10~40도 범위를 14픽셀 높이로 매핑합니다. |

**확장 아이디어:**
- 최고/최저 온도를 기록하여 함께 표시하기
- 온도 변화 추이를 OLED에 꺾은선 그래프로 표시하기
- LED로 상태별 색상 표시 (빨강=Hot, 파랑=Cold, 초록=Good)

#### PC 실시간 대시보드

Pico에서 시리얼로 출력하는 데이터를 PC에서 실시간 그래프로 시각화합니다. 온도와 습도를 동시에 표시하고, 쾌적 온도 영역(18~28도)을 배경색으로 구분합니다.

:::caution[사전 준비]
1. 터미널에서 라이브러리를 설치하세요:
   ```
   pip install pyserial matplotlib
   ```
   (Mac: `pip3 install pyserial matplotlib`)
2. **Thonny를 먼저 닫아주세요!** Thonny가 열려 있으면 시리얼 포트를 사용할 수 없습니다.
3. 포트 이름 확인: Windows는 `COM3` 등, Mac은 `/dev/tty.usbmodem...` 형태입니다.

```python
# PC 실시간 대시보드 — 교실 온도 대시보드
# 실행: python classroom_temp_pc.py

import serial
import matplotlib.pyplot as plt
from matplotlib.animation import FuncAnimation
from collections import deque
import time
import re

# ── 시리얼 연결 (포트 이름은 환경에 맞게 수정) ──
# Windows: 'COM3', macOS: '/dev/tty.usbmodem...', Linux: '/dev/ttyACM0'
PORT = '/dev/tty.usbmodem1101'
BAUD = 115200

ser = serial.Serial(PORT, BAUD, timeout=1)
time.sleep(2)  # 연결 안정화

# ── 데이터 버퍼 ──
MAX_POINTS = 100
times = deque(maxlen=MAX_POINTS)
temps = deque(maxlen=MAX_POINTS)
humis = deque(maxlen=MAX_POINTS)

# ── 그래프 설정 ──
plt.style.use('dark_background')
fig, ax1 = plt.subplots(figsize=(10, 5))
fig.suptitle('Classroom Temp Dashboard', fontsize=14, color='cyan')

ax2 = ax1.twinx()
line_temp, = ax1.plot([], [], 'r-o', markersize=3, label='Temp (°C)')
line_humi, = ax2.plot([], [], 'c-s', markersize=3, label='Humi (%)')

ax1.set_xlabel('Time (s)')
ax1.set_ylabel('Temperature (°C)', color='red')
ax2.set_ylabel('Humidity (%)', color='cyan')
ax1.set_ylim(10, 40)
ax2.set_ylim(0, 100)

# 쾌적 온도 영역 표시
ax1.axhspan(18, 28, alpha=0.15, color='green', label='Good (18-28°C)')
ax1.axhspan(0, 18, alpha=0.10, color='blue', label='Cold')
ax1.axhspan(28, 50, alpha=0.10, color='red', label='Hot')

ax1.legend(loc='upper left')
ax2.legend(loc='upper right')
ax1.grid(True, alpha=0.3)

# ── 실시간 업데이트 ──
def update(frame):
    try:
        line = ser.readline().decode('utf-8').strip()
        if not line:
            return
        # 파싱: "12s | 24.5°C 55.2% [Good]"
        m = re.match(r'(\d+)s \| ([\d.]+)°C ([\d.]+)% \[(.+)\]', line)
        if not m:
            return
        t = int(m.group(1))
        temp = float(m.group(2))
        humi = float(m.group(3))

        times.append(t)
        temps.append(temp)
        humis.append(humi)

        line_temp.set_data(list(times), list(temps))
        line_humi.set_data(list(times), list(humis))
        ax1.set_xlim(max(0, t - 200), t + 10)
        fig.canvas.manager.set_window_title(f'Classroom — {temp}°C {humi}% [{m.group(4)}]')
    except:
        pass

ani = FuncAnimation(fig, update, interval=2000, cache_frame_data=False)
plt.tight_layout()
plt.show()
```

실행하면 어두운 배경의 실시간 그래프 창이 열리고, Pico에서 2초마다 보내는 온도/습도 데이터가 그래프에 추가됩니다. 초록색 영역이 쾌적 온도(18~28도) 범위입니다.

### 프로젝트 2: 소음 레벨 모니터

**센서:** Grove 소리 센서 (ADC — A1 포트, GP27)

**아이디어:** 교실의 소음 수준을 실시간으로 측정하고, 7단 레벨 미터로 시각화합니다. "Quiet / Normal / Loud / V.Loud" 4단계로 판정합니다.

**OLED 대시보드 설계:**
```
+---------------------------+
|████ Noise Level ██████████|  ← 반전 헤더
|─────────────────────────--|
|          45%              |
|        [Normal]           |
|─────────────────────────--|
| ██ ██ ██ ░░ ░░ ░░ ░░     |  ← 7단 레벨 미터
+---------------------------+
```

**연결:** 소리 센서 → A1 포트(GP27), OLED → I2C1 포트

:::caution[포트 주의]
소리 센서는 반드시 **A1 포트(GP27)**에 연결하세요. A0 포트(GP26)는 빛 센서용입니다. 잘못 연결하면 값이 이상하게 나올 수 있습니다.
:::

#### 전체 코드

```python
from machine import ADC, Pin, I2C
import time

# ── 소리 센서 설정 (ADC — Grove A1 포트: GP27) ──
sound = ADC(Pin(27))

# ── OLED 설정 ──
i2c = I2C(1, sda=Pin(6), scl=Pin(7), freq=100000)
from ssd1306 import SSD1306_I2C
oled = SSD1306_I2C(128, 64, i2c)

def get_noise_level(value):
    pct = value / 65535 * 100
    if pct < 20:
        return pct, "Quiet", 1
    elif pct < 50:
        return pct, "Normal", 3
    elif pct < 75:
        return pct, "Loud", 5
    else:
        return pct, "V.Loud", 7

# ── CSV 파일 준비 ──
import os
try:
    size = os.stat("noise_log.csv")[6]
except:
    size = 0
f = open("noise_log.csv", "a")
if size == 0:
    f.write("time_s,noise_pct,level\n")
start = time.ticks_ms()

# ── 메인 루프 ──
while True:
    raw = sound.read_u16()
    pct, level_text, bars = get_noise_level(raw)
    elapsed = time.ticks_diff(time.ticks_ms(), start) // 1000

    # OLED 대시보드
    oled.fill(0)
    # 헤더 바
    oled.fill_rect(0, 0, 128, 12, 1)
    oled.text("Noise Level", 24, 2, 0)
    oled.hline(0, 13, 128, 1)
    # 퍼센트 크게 표시
    pct_str = f"{pct:.0f}%"
    oled.text(pct_str, 64 - len(pct_str) * 4, 18)
    oled.text(f"[{level_text}]", 64 - len(level_text) * 4 - 8, 30)
    # 레벨 미터 (막대 7칸, 2px 간격)
    oled.hline(0, 42, 128, 1)
    for i in range(7):
        x = 10 + i * 16
        if i < bars:
            oled.fill_rect(x, 46, 12, 16, 1)
        else:
            oled.rect(x, 46, 12, 16, 1)
    oled.show()

    # CSV 저장
    f.write(f"{elapsed},{pct:.1f},{level_text}\n")
    f.flush()

    print(f"{elapsed}s | {pct:.0f}% [{level_text}]")
    time.sleep(0.5)
```

#### 줄별 해설

| 줄 | 코드 | 설명 |
|----|------|------|
| 4 | `ADC(Pin(27))` | GP27(A1 포트)에서 아날로그 값을 읽습니다. ADC는 전압을 0~65535 사이의 숫자로 변환합니다. |
| 12-20 | `get_noise_level()` | 소음 퍼센트에 따라 4단계(Quiet/Normal/Loud/V.Loud)로 판정합니다. `bars`는 레벨 미터에 채울 막대 수입니다. |
| 28 | `read_u16()` | ADC에서 16비트 값(0~65535)을 읽습니다. 소리가 클수록 값이 커집니다. |
| 33 | `oled.fill_rect(0, 0, 128, 12, 1)` | 반전 헤더 바를 그립니다. 흰 배경 위에 검은 글씨로 "Noise Level"을 표시합니다. |
| 37-38 | 퍼센트 가운데 정렬 | `64 - len(pct_str) * 4`로 문자열 길이에 따라 중앙에 위치시킵니다. |
| 42-48 | 레벨 미터 그리기 | 7개의 막대를 그리는데, `bars` 수만큼 채운 막대(`fill_rect`), 나머지는 빈 테두리(`rect`)로 표시합니다. |

**확장 아이디어:**
- 일정 시간 동안 조용했던 누적 시간을 표시하기
- 소음이 기준치를 넘으면 LED로 경고하기

# 파형 (롤링)
line_wave, = ax_wave.plot([], [], 'lime', linewidth=1)
ax_wave.set_ylim(0, 100)
ax_wave.set_title('Noise Waveform', color='lime')

ax_wave.set_ylabel('Level (%)')
ax_wave.grid(True, alpha=0.3)

        noise_pcts.append(pct)

    except:
        pass

plt.tight_layout()

### 프로젝트 3: 자동 조명 측정기

**센서:** Grove 빛 센서 (ADC — A0 포트, GP26)

**아이디어:** 교실 밝기를 실시간으로 측정하고, OLED에 실시간 그래프를 그립니다. "Dark / Normal / Bright" 상태를 자동 판정합니다.

**OLED 대시보드 설계:**
```
+---------------------------+
|████ Light Meter ██████████|  ← 반전 헤더
|─────────────────────────--|
| 62%             [Normal]  |
| [████████████████░░░░░░]  |  ← 프로그레스 바
|─────────────────────────--|
| ┌────── sparkline ──────┐ |
| │ ╱╲  ╱╲╱╲              │ |  ← 영역 그래프
| └───────────────────────┘ |
+---------------------------+
```

**연결:** 빛 센서 → A0 포트(GP26), OLED → I2C1 포트

#### 전체 코드

```python
from machine import ADC, Pin, I2C
import time

# ── 빛 센서 설정 (ADC — Grove A0 포트: GP26) ──
light = ADC(Pin(26))

# ── OLED 설정 ──
i2c = I2C(1, sda=Pin(6), scl=Pin(7), freq=100000)
from ssd1306 import SSD1306_I2C
oled = SSD1306_I2C(128, 64, i2c)

# 그래프 데이터 저장 (최근 100개)
graph_data = []

def get_brightness(value):
    pct = value / 65535 * 100
    if pct < 20:
        return pct, "Dark"
    elif pct < 60:
        return pct, "Normal"
    else:
        return pct, "Bright"

# ── CSV 파일 준비 ──
import os
try:
    size = os.stat("light_log.csv")[6]
except:
    size = 0
f = open("light_log.csv", "a")
if size == 0:
    f.write("time_s,light_pct,status\n")
start = time.ticks_ms()

# ── 메인 루프 ──
while True:
    raw = light.read_u16()
    pct, status = get_brightness(raw)
    elapsed = time.ticks_diff(time.ticks_ms(), start) // 1000

    # 그래프 데이터 추가
    graph_data.append(int(pct))
    if len(graph_data) > 100:
        graph_data.pop(0)

    # OLED 대시보드
    oled.fill(0)
    # 헤더 바
    oled.fill_rect(0, 0, 128, 12, 1)
    oled.text("Light Meter", 24, 2, 0)
    oled.hline(0, 13, 128, 1)
    # 현재 값 + 상태
    oled.text(f"{pct:.0f}%", 4, 16)
    oled.text(f"[{status}]", 64, 16)
    # 막대그래프
    bar_w = int(min(pct, 100) / 100 * 108)
    oled.rect(8, 28, 112, 8, 1)
    oled.fill_rect(10, 30, bar_w, 4, 1)
    # 스파크라인 영역 (테두리 프레임)
    oled.hline(0, 38, 128, 1)
    oled.rect(0, 40, 128, 24, 1)
    for i in range(min(len(graph_data), 124)):
        idx = len(graph_data) - min(len(graph_data), 124) + i
        h = int(graph_data[idx] / 100 * 20)
        h = max(1, min(20, h))
        oled.vline(2 + i, 62 - h, h, 1)
    oled.show()

    # CSV 저장
    f.write(f"{elapsed},{pct:.1f},{status}\n")
    f.flush()

    print(f"{elapsed}s | {pct:.0f}% [{status}]")
    time.sleep(1)
```

#### 줄별 해설

| 줄 | 코드 | 설명 |
|----|------|------|
| 4 | `ADC(Pin(26))` | GP26(A0 포트)에서 밝기를 아날로그 값으로 읽습니다. |
| 13 | `graph_data = []` | 최근 100개의 밝기 값을 저장하는 리스트입니다. OLED 그래프를 그리는 데 사용합니다. |
| 33 | `graph_data.pop(0)` | 리스트가 100개를 넘으면 가장 오래된 값을 제거합니다. 슬라이딩 윈도우 방식입니다. |
| 41 | `oled.fill_rect(0, 0, 128, 12, 1)` | 반전 헤더 바를 그립니다. 흰 배경 위에 검은 글씨로 "Light Meter"를 표시합니다. |
| 48-50 | 프로그레스 바 | 현재 밝기를 테두리 있는 막대그래프로 표시합니다. 0~100%를 108픽셀로 매핑합니다. |
| 52-58 | 스파크라인 영역 | 테두리 프레임 안에 각 데이터 값을 세로선(`vline`)으로 그립니다. 최근 124개를 표시합니다. |

**확장 아이디어:**
- 밝기에 따라 Pico 내장 LED의 밝기를 조절하기 (PWM)
- 급격한 밝기 변화(예: 교실 불 켜고 끄기)를 감지하여 이벤트 기록
- 창가 vs 복도 쪽 밝기를 비교하는 실험

## 잠깐, 방금 무슨 일이 일어난 걸까요?

여러분이 방금 한 일을 다시 생각해보세요.

교실의 물리적 환경(온도, 소리, 빛)이
**→** 센서 내부에서 **전기 신호**로 바뀌고
**→** ADC 또는 I2C를 통해 **디지털 데이터**로 변환되고
**→** 코드가 그 데이터를 OLED 화면의 **시각적 대시보드**로 표현하고
**→** CSV 파일로 **기록**까지 자동으로 해주었습니다.

스마트폰의 날씨 앱, 공기청정기의 공기질 표시, 스마트워치의 심박수 화면 — 모두 이 원리의 확장입니다. 여러분은 방금 **IoT 장치의 기본 구조**를 직접 만든 것입니다.

## 트러블슈팅

### OLED에 아무것도 안 보여요
- I2C1 포트(SDA=GP6, SCL=GP7)에 연결되어 있는지 확인하세요.
- `ssd1306` 라이브러리가 Pico에 설치되어 있는지 확인하세요. Thonny의 패키지 관리자에서 `micropython-ssd1306`을 설치할 수 있습니다.
- OLED 주소가 0x3C인지 `i2c.scan()`으로 확인해보세요.

### 센서 값이 이상해요 (0이나 65535만 나옴)
- DHT20: I2C 포트에 연결했는지 확인하세요. ADC 포트에는 동작하지 않습니다.
- 빛/소리 센서: **반드시 ADC 포트**(A0 또는 A1)에 연결해야 합니다. 디지털 포트에 연결하면 값이 나오지 않습니다.
- 소리 센서가 A0에 연결되어 있다면 `Pin(26)`으로, A1에 연결되어 있다면 `Pin(27)`로 코드를 수정하세요.

### CSV 파일이 저장되지 않아요
- `f.flush()`가 있는지 확인하세요. 이 명령이 없으면 Pico가 꺼질 때 데이터가 손실됩니다.
- Pico의 저장 공간이 가득 찼을 수 있습니다. Thonny에서 Pico의 파일 목록을 확인하세요.
