### 센서 조합 시 주의사항

### 프로젝트 1: 스마트 환기 시스템

**센서:** DHT20 온습도 (I2C, 0x38) + SCD41 CO2 (I2C, 0x62) + OLED (I2C, 0x3C)

**아이디어:** CO2 농도와 온도를 종합 판단하여 환기 필요성을 자동으로 알려줍니다. CO2가 1500ppm 넘거나 온도가 높거나 습도가 80%를 넘으면 "VENT NOW!", 1000ppm 넘거나 온도가 높으면 "Warning"으로 단계별 경고를 표시합니다.

**스마트 판단 로직:**
```
CO2 > 1500 OR 온도 > 30°C OR 습도 > 80% → "VENT NOW!" (반전 표시)
CO2 > 1000 OR 온도 > 27°C → "Warning"
그 외 → "Good"
```

**OLED 대시보드 설계:**
```
+---------------------------+
|████ Smart Vent ███████████|  ← 반전 헤더
|─────────────────────────--|
| CO2 1234ppm               |
| C [████████████░░░░]      |  ← CO2 프로그레스 바
| T [████████░░░░░░░░] 25.3 |  ← 온도 프로그레스 바
| H [██████████░░░░░░] 58.2 |  ← 습도 프로그레스 바
|████ >> VENT NOW! << ██████|  ← 반전 경고
+---------------------------+
```

**연결:** 세 장치 모두 I2C 통신 — 같은 I2C1 버스에 연결 (주소: DHT20=0x38, SCD41=0x62, OLED=0x3C)

:::note[SCD41 센서 특성]
SCD41은 `start_periodic_measurement` 명령(0x21, 0xB1)을 **한 번만** 보내면, 이후 5초마다 자동으로 CO2를 측정합니다. 매 루프마다 시작 명령을 보내면 5초 지연이 반복되므로, 반드시 루프 밖에서 한 번만 호출하세요.
:::

#### 전체 코드

```python
from machine import I2C, Pin
import time

# ── I2C 설정 (DHT20 + SCD41 + OLED 공유) ──
i2c = I2C(1, sda=Pin(6), scl=Pin(7), freq=100000)
dht_addr = 0x38
scd_addr = 0x62
time.sleep(0.1)

from ssd1306 import SSD1306_I2C
oled = SSD1306_I2C(128, 64, i2c)

def read_dht20():
    i2c.writeto(dht_addr, bytes([0xAC, 0x33, 0x00]))
    time.sleep(0.08)
    data = i2c.readfrom(dht_addr, 7)
    humi = ((data[1] << 12) | (data[2] << 4) | (data[3] >> 4)) / 1048576 * 100
    temp = (((data[3] & 0x0F) << 16) | (data[4] << 8) | data[5]) / 1048576 * 200 - 50
    return round(temp, 1), round(humi, 1)

def read_scd41():
    # 데이터 읽기 (주기 측정 모드에서 자동 갱신됨)
    i2c.writeto(scd_addr, bytes([0xEC, 0x05]))
    time.sleep(0.01)
    data = i2c.readfrom(scd_addr, 9)
    co2 = (data[0] << 8) | data[1]
    return co2

# SCD41 주기 측정 시작 (한 번만 호출)
i2c.writeto(scd_addr, bytes([0x21, 0xB1]))
print("SCD41 준비 중... 5초 대기")
time.sleep(5)

def judge_ventilation(co2, temp, humi):
    if co2 > 1500 or temp > 30 or humi > 80:
        return "VENT NOW!", True
    elif co2 > 1000 or temp > 27:
        return "Warning", True
    else:
        return "Good", False

# ── 메인 루프 ──
while True:
    temp, humi = read_dht20()
    co2 = read_scd41()
    status, need_vent = judge_ventilation(co2, temp, humi)

    # OLED 대시보드
    oled.fill(0)
    # 헤더 바
    oled.fill_rect(0, 0, 128, 12, 1)
    oled.text("Smart Vent", 28, 2, 0)
    oled.hline(0, 13, 128, 1)
    # CO2 값 크게 표시
    oled.text(f"CO2 {co2}ppm", 4, 16)
    # 3개 미니 프로그레스 바
    # CO2 바 (0~2000ppm)
    oled.text("C", 0, 28)
    co2_w = int(min(co2, 2000) / 2000 * 80)
    oled.rect(10, 28, 82, 8, 1)
    oled.fill_rect(12, 30, co2_w, 4, 1)
    # Temp 바 (0~40C)
    oled.text("T", 0, 38)
    temp_w = int(min(max(temp, 0), 40) / 40 * 80)
    oled.rect(10, 38, 82, 8, 1)
    oled.fill_rect(12, 40, temp_w, 4, 1)
    oled.text(f"{temp:.0f}", 96, 38)
    # Humi 바 (0~100%)
    oled.text("H", 0, 48)
    humi_w = int(min(humi, 100) / 100 * 80)
    oled.rect(10, 48, 82, 8, 1)
    oled.fill_rect(12, 50, humi_w, 4, 1)
    oled.text(f"{humi:.0f}", 96, 48)
    # 상태 표시
    if need_vent:
        oled.fill_rect(0, 56, 128, 8, 1)
        oled.text(f">> {status} <<", 4, 56, 0)
    else:
        oled.text(f"[{status}]", 80, 16)
    oled.show()

    print(f"CO2:{co2}ppm T:{temp}C H:{humi}% [{status}]")
    time.sleep(5)
```

| 4-7 | I2C 설정 | DHT20(0x38), SCD41(0x62), OLED(0x3C)이 같은 I2C1 버스를 공유합니다. 주소가 모두 다르므로 충돌하지 않습니다. |

```python
# PC 실시간 대시보드 — 스마트 환기 시스템
# 실행: python smart_vent_pc.py

import serial
import matplotlib.pyplot as plt
from matplotlib.animation import FuncAnimation
from matplotlib.patches import FancyBboxPatch
from collections import deque
import time
import re

# ── 시리얼 연결 ──
PORT = '/dev/tty.usbmodem1101'
BAUD = 115200

ser = serial.Serial(PORT, BAUD, timeout=1)
time.sleep(2)

# ── 데이터 버퍼 ──
MAX_POINTS = 100
idx_buf = deque(maxlen=MAX_POINTS)
co2_buf = deque(maxlen=MAX_POINTS)
temp_buf = deque(maxlen=MAX_POINTS)
humi_buf = deque(maxlen=MAX_POINTS)
count = 0

# ── 그래프 설정 (2x2) ──
plt.style.use('dark_background')
fig, ((ax_co2, ax_temp), (ax_humi, ax_status)) = plt.subplots(2, 2, figsize=(12, 8))
fig.suptitle('Smart Ventilation Dashboard', fontsize=14, color='cyan')

# CO2 트렌드
line_co2, = ax_co2.plot([], [], 'r-', linewidth=2)
ax_co2.set_title('CO2 (ppm)', color='red')
ax_co2.set_ylim(300, 2500)
ax_co2.axhline(y=1000, color='orange', linestyle='--', alpha=0.5, label='Warning')
ax_co2.axhline(y=1500, color='red', linestyle='--', alpha=0.5, label='Vent Now!')
ax_co2.legend(fontsize=8)
ax_co2.grid(True, alpha=0.3)

# 온도 트렌드
line_temp, = ax_temp.plot([], [], '#ff8800', linewidth=2)
ax_temp.set_title('Temperature (°C)', color='#ff8800')
ax_temp.set_ylim(10, 40)
ax_temp.axhspan(18, 28, alpha=0.1, color='green')
ax_temp.grid(True, alpha=0.3)

# 습도 트렌드
line_humi, = ax_humi.plot([], [], 'cyan', linewidth=2)
ax_humi.set_title('Humidity (%)', color='cyan')
ax_humi.set_ylim(0, 100)
ax_humi.axhspan(40, 60, alpha=0.1, color='green')
ax_humi.grid(True, alpha=0.3)

# 상태 표시
ax_status.set_xlim(0, 10)
ax_status.set_ylim(0, 10)
ax_status.set_title('Status', color='white')
ax_status.axis('off')
status_text = ax_status.text(5, 5, 'Waiting...', ha='center', va='center',
                              fontsize=24, color='gray',
                              bbox=dict(boxstyle='round,pad=0.5', facecolor='#222'))

def update(frame):
    global count
    try:
        line = ser.readline().decode('utf-8').strip()
        if not line:
            return
        # 파싱: "CO2:850ppm T:24.5C H:55.2% [Good]"
        m = re.match(r'CO2:(\d+)ppm T:([\d.]+)C H:([\d.]+)% \[(.+)\]', line)
        if not m:
            return
        co2 = int(m.group(1))
        temp = float(m.group(2))
        humi = float(m.group(3))
        status = m.group(4)
        count += 1

        idx_buf.append(count)
        co2_buf.append(co2)
        temp_buf.append(temp)
        humi_buf.append(humi)

        x = list(idx_buf)
        line_co2.set_data(x, list(co2_buf))
        line_temp.set_data(x, list(temp_buf))
        line_humi.set_data(x, list(humi_buf))

        for ax in [ax_co2, ax_temp, ax_humi]:
            ax.set_xlim(max(0, count - MAX_POINTS), count + 5)

        # 상태 색상
        colors = {'Good': '#00ff88', 'Warning': '#ffaa00', 'VENT NOW!': '#ff3333'}
        c = colors.get(status, 'gray')
        status_text.set_text(f'{status}\nCO2: {co2} ppm')
        status_text.set_color(c)

        fig.canvas.manager.set_window_title(f'Vent — CO2:{co2} T:{temp} H:{humi} [{status}]')
    except:
        pass

ani = FuncAnimation(fig, update, interval=5000, cache_frame_data=False)
plt.tight_layout()
plt.show()
```

### 프로젝트 2: 간이 보안 시스템

**센서:** 초음파 거리 센서 (Digital, GP16+GP17) + 소리 센서 (ADC, GP27) + 부저 (PWM, GP18) + OLED

**아이디어:** 거리 감지와 소음 감지를 조합하여 침입을 판별합니다. 거리가 가까워지면서 소리도 나면 "INTRUDER!" 경보, 둘 중 하나만이면 "ALERT!" 주의, 둘 다 아니면 "Safe"입니다.

**스마트 판단 로직:**
```
거리 < 50cm AND 소음 > 40% → "INTRUDER!" (부저 강하게)
거리 < 50cm OR 소음 > 40%  → "ALERT!" (부저 약하게)
그 외 → "Safe" (부저 꺼짐)
```

:::caution[핀 충돌 해결]
초음파 센서가 GP16(TRIG)과 GP17(ECHO)을 사용합니다. 부저의 기본 핀도 GP16이므로, 이 프로젝트에서는 부저를 **GP18**에 연결합니다.
:::

#### 전체 코드

```python
from machine import Pin, ADC, PWM, I2C, time_pulse_us
import time

# ── 초음파 센서 설정 ──
trig = Pin(16, Pin.OUT)
echo = Pin(17, Pin.IN)

# ── 소리 센서 설정 (Grove A1: GP27) ──
sound = ADC(Pin(27))

# ── 부저 설정 (GP18 — 초음파가 GP16 사용하므로 대체 핀) ──
buzzer = PWM(Pin(18))
buzzer.duty_u16(0)

# ── OLED 설정 ──
i2c = I2C(1, sda=Pin(6), scl=Pin(7), freq=100000)
from ssd1306 import SSD1306_I2C
oled = SSD1306_I2C(128, 64, i2c)

DIST_THRESHOLD = 50    # cm 이내 감지
SOUND_THRESHOLD = 40   # % 이상 감지

def read_distance():
    trig.value(0)
    time.sleep_us(5)
    trig.value(1)
    time.sleep_us(10)
    trig.value(0)
    duration = time_pulse_us(echo, 1, 30000)
    if duration < 0:
        return 999
    return round(duration / 58, 1)

def read_sound():
    return sound.read_u16() / 65535 * 100

def alarm_on(level):
    freq = 1000 + level * 500
    buzzer.freq(freq)
    buzzer.duty_u16(32768)

def alarm_off():
    buzzer.duty_u16(0)

# ── 메인 루프 ──
while True:
    dist = read_distance()
    noise = read_sound()

    # 침입 판단: 거리 가까움 AND 소리 감지
    dist_alert = dist < DIST_THRESHOLD
    sound_alert = noise > SOUND_THRESHOLD

    if dist_alert and sound_alert:
        status = "INTRUDER!"
        alarm_on(3)
    elif dist_alert or sound_alert:
        status = "ALERT!"
        alarm_on(1)
    else:
        status = "Safe"
        alarm_off()

    # OLED 대시보드
    oled.fill(0)
    # 헤더 바
    oled.fill_rect(0, 0, 128, 12, 1)
    oled.text("Security", 32, 2, 0)
    oled.hline(0, 13, 128, 1)
    # 거리 표시 + 근접 인디케이터
    oled.text(f"Dist: {dist:.0f}cm", 4, 16)
    prox = min(int((DIST_THRESHOLD - min(dist, DIST_THRESHOLD)) / DIST_THRESHOLD * 5), 5)
    for i in range(5):
        x = 100 + i * 5
        if i < prox:
            oled.fill_rect(x, 16, 4, 8, 1)
        else:
            oled.rect(x, 16, 4, 8, 1)
    # 소음 레벨 바
    oled.text(f"Sound:{noise:.0f}%", 4, 28)
    noise_w = int(min(noise, 100) / 100 * 44)
    oled.rect(80, 28, 46, 8, 1)
    oled.fill_rect(81, 29, noise_w, 6, 1)
    # 상태 표시
    oled.hline(0, 40, 128, 1)
    if status == "INTRUDER!":
        oled.fill_rect(0, 42, 128, 22, 1)
        oled.text("!! INTRUDER !!", 8, 48, 0)
    elif status == "ALERT!":
        oled.fill_rect(4, 44, 8, 8, 1)
        oled.text("ALERT!", 16, 44)
        oled.text("Check area", 16, 54)
    else:
        oled.rect(4, 44, 8, 8, 1)
        oled.text("Safe", 16, 44)
        oled.text("Monitoring...", 16, 54)
    oled.show()

    print(f"D:{dist:.0f}cm N:{noise:.0f}% [{status}]")
    time.sleep(0.3)
```

### 프로젝트 3: 종합 쾌적도 대시보드

**센서:** DHT20 온습도 (I2C) + 빛 센서 (ADC, GP26) + 소리 센서 (ADC, GP27) + OLED

**아이디어:** 온도, 습도, 밝기, 소음 4가지 환경 요소를 동시에 측정하고, 각 항목의 점수를 계산하여 종합 쾌적도(0~100점)를 표시합니다.

**점수 계산 로직:**
```python
# 각 항목 0~100점, 4개 평균 = 종합 쾌적도
temp_score:  20~26°C → 100점, 18~28°C → 70점, 나머지 → 30점
humi_score:  40~60% → 100점, 30~70% → 70점, 나머지 → 30점
light_score: 30~70% → 100점, 나머지 → 50점
noise_score: 100 - noise_pct (낮을수록 좋음)
```

**OLED 대시보드 설계:**
```
+---------------------------+
|████ Comfort ██████████████|  ← 반전 헤더
|─────────────────────────--|
|  T    H    L    N         |
| ┌──┐ ┌──┐ ┌──┐ ┌──┐      |  ← 4개 세로 바
| |██| |██| |██| |██|      |
| └──┘ └──┘ └──┘ └──┘      |
|─────────────────────────--|
| Score:84/100              |
| [████████████████░░░░░░]  |  ← 프로그레스 바
+---------------------------+
```

#### 전체 코드

```python
from machine import I2C, Pin, ADC
import time

# ── 센서 설정 ──
i2c = I2C(1, sda=Pin(6), scl=Pin(7), freq=100000)
dht_addr = 0x38
light = ADC(Pin(26))  # Grove A0 포트
sound = ADC(Pin(27))  # Grove A1 포트
time.sleep(0.1)

from ssd1306 import SSD1306_I2C
oled = SSD1306_I2C(128, 64, i2c)

def read_dht20():
    i2c.writeto(dht_addr, bytes([0xAC, 0x33, 0x00]))
    time.sleep(0.08)
    data = i2c.readfrom(dht_addr, 7)
    humi = ((data[1] << 12) | (data[2] << 4) | (data[3] >> 4)) / 1048576 * 100
    temp = (((data[3] & 0x0F) << 16) | (data[4] << 8) | data[5]) / 1048576 * 200 - 50
    return round(temp, 1), round(humi, 1)

def calc_comfort(temp, humi, light_pct, noise_pct):
    # 온도 점수 (20~26도가 최적)
    if 20 <= temp <= 26:
        t_score = 100
    elif 18 <= temp <= 28:
        t_score = 70
    else:
        t_score = 30

    # 습도 점수 (40~60%가 최적)
    if 40 <= humi <= 60:
        h_score = 100
    elif 30 <= humi <= 70:
        h_score = 70
    else:
        h_score = 30

    # 조도 점수 (30~70%가 최적)
    if 30 <= light_pct <= 70:
        l_score = 100
    else:
        l_score = 50

    # 소음 점수 (낮을수록 좋음)
    n_score = max(0, 100 - noise_pct)

    total = int((t_score + h_score + l_score + n_score) // 4)
    return total

# ── CSV 파일 준비 ──
import os
try:
    size = os.stat("comfort_log.csv")[6]
except:
    size = 0
f = open("comfort_log.csv", "a")
if size == 0:
    f.write("time_s,temp,humi,light,noise,score\n")
start = time.ticks_ms()

# ── 메인 루프 ──
while True:
    temp, humi = read_dht20()
    light_pct = light.read_u16() / 65535 * 100
    noise_pct = sound.read_u16() / 65535 * 100
    score = calc_comfort(temp, humi, light_pct, noise_pct)
    elapsed = time.ticks_diff(time.ticks_ms(), start) // 1000

    # 개별 점수 계산 (대시보드용)
    t_sc = 100 if 20 <= temp <= 26 else (70 if 18 <= temp <= 28 else 30)
    h_sc = 100 if 40 <= humi <= 60 else (70 if 30 <= humi <= 70 else 30)
    l_sc = 100 if 30 <= light_pct <= 70 else 50
    n_sc = max(0, int(100 - noise_pct))

    # OLED 대시보드
    oled.fill(0)
    # 헤더 바
    oled.fill_rect(0, 0, 128, 12, 1)
    oled.text("Comfort", 36, 2, 0)
    oled.hline(0, 13, 128, 1)
    # 4개 미니 바 (T/H/L/N 나란히)
    labels = ['T', 'H', 'L', 'N']
    scores = [t_sc, h_sc, l_sc, n_sc]
    for i in range(4):
        x = 4 + i * 32
        oled.text(labels[i], x + 4, 16)
        bar_h = int(scores[i] / 100 * 18)
        oled.rect(x, 25, 12, 20, 1)
        oled.fill_rect(x + 1, 44 - bar_h, 10, bar_h, 1)
    # 종합 점수
    oled.hline(0, 47, 128, 1)
    oled.text(f"Score:{score}/100", 4, 50)
    # 점수 프로그레스 바 (테두리)
    oled.rect(4, 58, 120, 6, 1)
    oled.fill_rect(5, 59, min(score, 100) * 118 // 100, 4, 1)
    oled.show()

    # CSV 저장
    f.write(f"{elapsed},{temp},{humi},{light_pct:.0f},{noise_pct:.0f},{score}\n")
    f.flush()

    print(f"T:{temp} H:{humi} L:{light_pct:.0f} N:{noise_pct:.0f} => {score}점")
    time.sleep(2)
```

```python
                            ha='right', va='top', fontsize=16, color='cyan')

fig.suptitle('Comfort Dashboard', fontsize=14, color='cyan')

def calc_sub_scores(temp, humi, light, noise):
    t = 100 if 20 <= temp <= 26 else (70 if 18 <= temp <= 28 else 30)
    h = 100 if 40 <= humi <= 60 else (70 if 30 <= humi <= 70 else 30)
    l = 100 if 30 <= light <= 70 else 50
    n = max(0, 100 - noise)
    return {'T': t, 'H': h, 'L': l, 'N': int(n)}

def update(frame):
    global count
    try:
        line = ser.readline().decode('utf-8').strip()
        if not line:
            return
        # 파싱: "T:24.5 H:55.2 L:45 N:20 => 82점"
        m = re.match(r'T:([\d.]+) H:([\d.]+) L:(\d+) N:(\d+) => (\d+)', line)
        if not m:
            return
        temp = float(m.group(1))
        humi = float(m.group(2))
        light = float(m.group(3))
        noise = float(m.group(4))
        score = int(m.group(5))
        count += 1

        idx_buf.append(count)
        score_buf.append(score)

        subs = calc_sub_scores(temp, humi, light, noise)

        # 게이지 업데이트

        x = list(idx_buf)
        line_score.set_data(x, list(score_buf))
        ax_trend.set_xlim(max(0, count - MAX_POINTS), count + 5)
        score_text.set_text(f'{score}/100')

        fig.canvas.manager.set_window_title(f'Comfort — T:{temp} H:{humi} L:{light:.0f} N:{noise:.0f} => {score}')
    except:
        pass

ani = FuncAnimation(fig, update, interval=2000, cache_frame_data=False)
plt.tight_layout()
```

## 잠깐, 방금 무슨 일이 일어난 걸까요?

Lv.1에서는 센서 **하나**의 데이터를 읽어 대시보드에 표시했습니다.

Lv.2에서는 센서 **여러 개**의 데이터를 **종합적으로 판단**하는 시스템을 만들었습니다.

이 차이는 생각보다 큽니다:
- "온도가 28도다" → **정보**
- "온도가 28도이고 CO2가 1200ppm이다 → 환기 필요!" → **지능적 판단**

여러분이 만든 시스템은 단순히 숫자를 보여주는 것이 아니라, **여러 데이터를 조합하여 의미 있는 결론을 내리는** 시스템입니다. 이것이 바로 "스마트"의 핵심이에요.

실제 스마트 홈, 스마트 빌딩, 자율주행차 — 모두 이 원리를 대규모로 확장한 것입니다.

## 바이브 코딩 활용 전략

Lv.2부터는 바이브 코딩이 핵심 도구입니다. 효과적인 프롬프트 작성법을 익혀봅시다.

### 좋은 프롬프트의 3요소

1. **맥락**: 어떤 하드웨어를 사용하는지 (Pico 2 WH, Grove Shield, I2C1)
2. **구체적 요청**: 어떤 기능을 만들고 싶은지 (CO2 + 온도 → 환기 판단)
3. **제약 조건**: 센서 주소, 핀 번호, OLED 크기 등

### 프롬프트 예시

```
나는 Pico 2 WH에 Grove Shield를 사용하고 있어.
I2C1(SDA=GP6, SCL=GP7)에 다음 장치들을 연결했어:
- SCD41 CO2 센서 (주소 0x62)
- DHT20 온습도 센서 (주소 0x38)
- SSD1306 OLED (주소 0x3C)

SCD41의 start_periodic_measurement 명령은 bytes([0x21, 0xB1])이고,
read_measurement 명령은 bytes([0xEC, 0x05])야.
시작 명령은 한 번만 보내고, 이후 5초마다 읽기만 하면 돼.

CO2가 1000ppm 넘으면 환기 경고를 OLED에 반전 표시해주는
MicroPython 코드를 만들어줘.
```

[Pico Makers](https://pico-makers.vercel.app)에서 Course B 모드를 선택하면, 각 레벨의 프로젝트 예시와 센서별 배선 정보를 확인할 수 있어요. 센서를 추가한 뒤 "통합 코드"를 AI 프롬프트에 함께 붙여넣으면 더 정확한 결과를 얻을 수 있습니다.

## 트러블슈팅

### SCD41에서 CO2 값이 0이 나와요
- `start_periodic_measurement` 명령(`bytes([0x21, 0xB1])`)을 보냈는지 확인하세요.
- 시작 명령 후 **5초를 기다린 뒤** 읽기를 시작해야 합니다.
- `i2c.scan()`으로 0x62가 보이는지 확인하세요.

### 부저에서 소리가 안 나요
- 초음파 센서와 함께 사용할 때, 부저 핀이 **GP18**인지 확인하세요 (GP16이 아님).
- `buzzer.duty_u16(0)`이면 소리가 꺼진 상태입니다. `duty_u16(32768)` 이상이어야 들립니다.

### 센서 값이 서로 간섭하는 것 같아요
- I2C 센서들은 주소가 다르면 간섭하지 않습니다. `i2c.scan()`으로 모든 장치가 보이는지 확인하세요.
- ADC 센서(빛, 소리)는 핀이 다르므로 간섭하지 않지만, 케이블이 너무 가까우면 노이즈가 생길 수 있습니다.

## 한 단계 더: Google Sheets 연동

지금까지 CSV 파일로 데이터를 저장했습니다. CSV는 Pico 내부에만 저장되기 때문에, USB를 연결해야 파일을 꺼낼 수 있었죠. 하지만 **Google Sheets로 데이터를 전송**하면, 교실 밖 어디서든 실시간으로 데이터를 확인할 수 있습니다.

:::tip[이 섹션은 선택 사항입니다]
Google Sheets 연동은 필수가 아닙니다. 하지만 교실 밖에서 전시하거나, 장기간 데이터를 수집하는 프로젝트에서는 매우 유용합니다. **Pico 2 WH**의 WiFi 기능을 활용합니다.
:::

### 원리

```
Pico 2 WH → WiFi로 인터넷 연결 → Google Apps Script(웹훅)에 POST → 스프레드시트에 자동 기록
```

Pico가 센서 데이터를 JSON 형태로 만들어서, Google이 제공하는 웹 주소(URL)로 보냅니다. Google Apps Script가 이 데이터를 받아서 스프레드시트에 한 줄씩 추가합니다.

:::note[처음 보는 단어가 있나요? 걱정 마세요!]
이 섹션에서 사용하는 용어를 쉽게 설명할게요:

- **JSON**: 데이터를 정해진 형태로 포장하는 방식이에요. 택배를 보낼 때 상자에 담아서 보내는 것처럼, 데이터도 형식에 맞게 포장해서 보냅니다. 예: `{"temp": 24.5, "humi": 55}`
- **POST**: 데이터를 서버(다른 컴퓨터)로 **보내는** 방법이에요. 편지를 우체통에 넣는 것과 비슷해요. (반대로 데이터를 **가져오는** 건 GET이라고 해요)
- **웹훅(Webhook)**: 특정 이벤트가 발생하면 자동으로 데이터를 보내주는 장치예요. "온도가 측정될 때마다 자동으로 구글에 알려줘!"라고 설정해두는 거예요.
- **URL**: 인터넷 주소예요. 네이버 주소(naver.com)처럼, Google Apps Script에도 고유한 주소가 있어요.
:::

### WiFi 연결 준비하기

인터넷으로 데이터를 보내려면 먼저 Pico를 WiFi에 연결해야 합니다.

:::tip[WiFi 이름(SSID)과 비밀번호 찾기]
**WiFi 이름(SSID)이란?**
핸드폰에서 WiFi를 연결할 때 목록에 보이는 이름과 같아요. 예: `school_wifi`, `iptime_5G` 등

**비밀번호는 어떻게 알 수 있나요?**
- **방법 1**: 선생님께 교실 WiFi 이름과 비밀번호를 물어보세요
- **방법 2**: 내 핸드폰 핫스팟을 사용하세요 (아래 설명 참고)
:::

:::caution[Pico는 2.4GHz WiFi만 됩니다!]
WiFi에는 2.4GHz와 5GHz 두 종류가 있어요. **Pico는 2.4GHz만 지원**합니다.

WiFi 이름 끝에 `_5G`가 붙어 있으면 5GHz이므로 연결되지 않아요!
- `school_wifi_5G` → 안 됨
- `school_wifi` 또는 `school_wifi_2.4G` → 사용 가능
:::

:::tip[학교 WiFi가 안 될 때: 핸드폰 핫스팟 사용하기]
학교 WiFi가 연결되지 않거나 비밀번호를 모르면, 내 핸드폰을 WiFi 공유기처럼 쓸 수 있어요!

**아이폰:**
설정 → 개인용 핫스팟 → "다른 사람의 접속 허용" 켜기

**안드로이드:**
설정 → 연결 → 모바일 핫스팟 → 핫스팟 켜기

켜면 핸드폰에 WiFi 이름과 비밀번호가 표시돼요. 그걸 코드에 넣으면 됩니다!

**주의:** 핫스팟은 모바일 데이터를 사용하므로, 수업 시간에만 잠깐 켜세요.
:::

### Step 1: Google 스프레드시트 + Apps Script 설정

아래 단계를 하나씩 따라 해보세요. 천천히 하면 누구나 할 수 있어요!

**① Google Sheets에서 Apps Script 열기**

1. 컴퓨터에서 인터넷 브라우저(크롬 등)를 열고, [Google Sheets](https://sheets.google.com)에 접속하세요. (구글 로그인이 필요해요)
2. **새 스프레드시트**를 만드세요.
3. 스프레드시트 메뉴에서 **확장 프로그램 → Apps Script**를 클릭하세요. (새 탭이 열립니다)

:::caution[중요: Google Sheets에서 시작하세요!]
Apps Script는 반드시 **Google Sheets 안에서** 만들어야 합니다:
1. Google Sheets에서 새 스프레드시트 만들기
2. 확장 프로그램 → Apps Script 클릭
이렇게 해야 `getActiveSpreadsheet()`가 정상 작동합니다.

만약 Google Drive에서 직접 Apps Script를 만들었다면, 코드의 첫 줄을 다음으로 교체하세요:
`SpreadsheetApp.openById("스프레드시트_ID")` (URL에서 /d/ 뒤의 긴 문자열)
:::

:::note[다른 방법: Google Drive에서 열기]
Google Drive(drive.google.com)에서 **"+ 새로 만들기" → "더보기" → "Google Apps Script"**로도 열 수 있지만, 이 방법은 스프레드시트와 연결되지 않은 독립(standalone) 스크립트가 만들어집니다. 반드시 위의 Google Sheets 방법을 먼저 시도하세요.
:::

**② 코드 붙여넣기**

4. 새로 열린 Apps Script 화면에 `function myFunction() { }` 라는 기본 코드가 보일 거예요.
5. **Ctrl+A** (맥은 Cmd+A)를 눌러 기본 코드를 **전체 선택**하세요.
6. **Delete** 또는 **Backspace** 키를 눌러 **삭제**하세요.
7. 아래의 "우리가 사용할 코드"를 **복사(Ctrl+C)**한 뒤, Apps Script 화면에 **붙여넣기(Ctrl+V)**하세요.
8. **Ctrl+S** (맥은 Cmd+S)를 눌러 **저장**하세요. (또는 위쪽의 💾 아이콘 클릭)

```javascript
function doPost(e) {
  var sheet = SpreadsheetApp.getActiveSpreadsheet()
    ? SpreadsheetApp.getActiveSpreadsheet().getActiveSheet()
    : SpreadsheetApp.openById("시트_ID를_여기에_붙여넣으세요").getActiveSheet();
  var data = JSON.parse(e.postData.contents);
  var row = [new Date()];
  var keys = Object.keys(data);
  if (sheet.getLastRow() === 0) {
    sheet.appendRow(["timestamp", ...keys]);
  }
  keys.forEach(function(key) { row.push(data[key]); });
  sheet.appendRow(row);
  return ContentService.createTextOutput(JSON.stringify({status: "ok"}))
    .setMimeType(ContentService.MimeType.JSON);
}
```

### Step 2: Pico 코드에 WiFi + 전송 기능 추가

기존 프로젝트 코드의 **맨 위**에 WiFi 연결 코드를 추가하고, 메인 루프 안에서 데이터를 전송합니다.

:::tip[urequests 라이브러리 설치하기]
아래 코드에서 사용하는 `urequests`는 Pico에 기본 설치되어 있지 않을 수 있어요. 먼저 설치해야 합니다!

1. Thonny에서 상단 메뉴 **도구** → **패키지 관리** 클릭
2. 검색창에 **`micropython-urequests`** 입력
3. 검색 결과에서 **`micropython-urequests`**를 클릭 → **설치** 버튼 클릭
4. "설치 완료"가 뜨면 창을 닫으세요

이 과정은 **한 번만** 하면 됩니다. 이미 설치했다면 다시 할 필요 없어요.
:::

```python
import network
import urequests
import time

# ── WiFi 연결 ──
# 아래 "WiFi이름"과 "WiFi비밀번호"를 여러분의 WiFi 정보로 바꾸세요!
# 예: wlan.connect("iptime_2G", "password1234")
wlan = network.WLAN(network.STA_IF)
wlan.active(True)
wlan.connect("WiFi이름", "WiFi비밀번호")

print("WiFi 연결 중...", end="")
timeout = 20  # 최대 20초 동안 연결 시도
while not wlan.isconnected() and timeout > 0:
    print(".", end="")
    time.sleep(1)
    timeout -= 1

if wlan.isconnected():
    print("\nWiFi 연결 완료!", wlan.ifconfig()[0])
else:
    print("\n⚠ WiFi 연결 실패!")
    print("  → WiFi 이름과 비밀번호를 확인하세요")
    print("  → 2.4GHz WiFi인지 확인하세요 (5GHz는 안 됩니다)")
    print("  → 오프라인 모드로 작동합니다 (데이터 전송 안 됨)")

# ── Google Sheets 전송 설정 ──
# ★ 아래 URL을 통째로 지우고, 아까 복사한 배포 URL을 붙여넣으세요! ★
# 올바른 형태 예시: "https://script.google.com/macros/s/AKfycbx.../exec"
SHEET_URL = "https://script.google.com/macros/s/여기에_배포_URL을_붙여넣으세요/exec"

def send_to_sheets(data_dict):
    """센서 데이터를 Google Sheets로 전송합니다."""
    if not wlan.isconnected():
        return False
    try:
        res = urequests.post(
            SHEET_URL,
            json=data_dict,
            headers={"Content-Type": "application/json"}
        )
        # Google Apps Script은 302 리다이렉트를 반환합니다
        if res.status_code in (301, 302):
            redirect_url = res.headers.get("Location", "")
            res.close()
            if redirect_url:
                res = urequests.post(
                    redirect_url,
                    json=data_dict,
                    headers={"Content-Type": "application/json"}
                )
        print(f"전송 완료: {res.status_code}")
        res.close()
        return True
    except Exception as e:
        print(f"전송 실패: {e}")
        return False
```

:::tip[Google Apps Script 리다이렉트]
Google Apps Script 웹 앱은 요청을 받으면 302 리다이렉트로 응답합니다.
위 코드는 리다이렉트를 자동으로 따라가도록 처리되어 있어요.
만약 데이터가 시트에 안 나타나면, Thonny 콘솔에서 상태 코드를 확인하세요.
:::

:::caution[URL 붙여넣기 주의!]
`SHEET_URL = "..."` 줄에서, 따옴표(`"`) 안의 내용을 **통째로 지우고** 아까 복사한 배포 URL **전체**를 붙여넣으세요.

올바른 예:
```python
SHEET_URL = "https://script.google.com/macros/s/AKfycbxAbCdEfG.../exec"
```

잘못된 예 (원래 텍스트가 남아 있으면 안 됩니다):
```python
SHEET_URL = "https://script.google.com/macros/s/여기에_배포_URL을_붙여넣으세요/exec"
```
:::

### Step 3: 메인 루프에서 전송하기

기존 메인 루프의 `print()` 줄 근처에 한 줄만 추가하면 됩니다.

```python
# 메인 루프 안에서 (기존 코드 유지)
# ... 센서 읽기, OLED 표시, CSV 저장 등 ...

# Google Sheets 전송 (추가)
send_to_sheets({
    "temp": temp,
    "humi": humi,
    "light": light_pct,
    "noise": noise_pct,
    "score": score
})

time.sleep(5)  # 너무 자주 보내지 않도록 간격 조절
```

:::note[전송 간격 주의]
Google Apps Script는 무료 계정 기준 하루 실행 횟수에 제한이 있습니다. **5초 이상 간격**으로 전송하면 수업 시간 동안 충분히 사용할 수 있습니다.
:::

### 결과 확인

전송이 성공하면, Google 스프레드시트에 이런 식으로 데이터가 쌓입니다:

```
| timestamp           | temp | humi | light | noise | score |
|---------------------|------|------|-------|-------|-------|
| 2026-03-14 09:30:01 | 24.2 | 55.3 | 62    | 25    | 84    |
| 2026-03-14 09:30:06 | 24.3 | 55.1 | 61    | 28    | 82    |
| ...                 | ...  | ...  | ...   | ...   | ...   |
```

:::tip[데이터가 잘 들어오는지 확인하세요!]
Pico에서 코드를 실행한 뒤, **Google Sheets 탭으로 돌아가서** 페이지를 새로고침(F5)하세요.

- 첫 번째 행에 **timestamp, temp, humi** 등의 컬럼 이름이 자동으로 생성됩니다
- 두 번째 행부터 센서 데이터가 한 줄씩 추가됩니다
- 5초마다 새로운 데이터가 들어오는 것을 확인할 수 있어요

아무것도 안 나온다면? 아래 "문제 해결" 섹션을 확인하세요!
:::

Google Sheets의 **차트 기능**을 사용하면 시간에 따른 변화를 그래프로 바로 볼 수 있어요!

### 문제 해결 (Google Sheets 연동)

| 증상 | 원인과 해결 방법 |
|------|-----------------|
| **"WiFi 연결 실패!"가 뜨고 점(.)만 계속 찍혀요** | WiFi 이름(SSID)과 비밀번호를 정확히 입력했는지 확인하세요. 대소문자를 구분합니다! 2.4GHz WiFi인지 확인하세요 (5GHz는 안 됩니다). 그래도 안 되면 핸드폰 핫스팟을 사용해보세요. |
| **"전송 실패"가 뜨거나 데이터가 안 보내져요** | `SHEET_URL`에 배포 URL을 제대로 붙여넣었는지 확인하세요. URL이 `https://script.google.com/macros/s/`로 시작하고 `/exec`로 끝나야 합니다. 인터넷 연결이 끊어졌을 수도 있으니 WiFi 상태도 확인하세요. |
| **스프레드시트에 데이터가 안 나와요** | Apps Script 배포가 제대로 됐는지 확인하세요. "배포 관리"에서 상태가 "활성"인지 확인하세요. 시트 이름이 기본값("Sheet1" 또는 "시트1")인지도 확인하세요. 시트 이름을 바꿨다면 코드가 첫 번째 시트를 사용하므로 괜찮습니다. |
| **`ImportError: no module named 'urequests'`** | Thonny에서 **도구 → 패키지 관리**를 열고, `micropython-urequests`를 검색하여 설치하세요. Pico가 인터넷에 연결되어 있어야 패키지를 설치할 수 있습니다. |
| **"이 앱은 확인되지 않았습니다" 경고에서 진행이 안 돼요** | "고급" → "(안전하지 않음)으로 이동" → "허용"을 순서대로 클릭하세요. 이 경고는 정상이에요. |
| **WiFi는 연결됐는데 전송만 안 돼요** | 학교 네트워크가 외부 접속을 차단할 수 있어요. 핸드폰 핫스팟으로 바꿔서 시도해보세요. |
