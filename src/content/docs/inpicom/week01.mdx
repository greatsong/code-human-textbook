:::caution[MQ2 예열 시간]
MQ2 센서는 안에 히터가 있어서 처음 켜면 **예열 시간이 필요**합니다. 1-2분 정도 지나야 값이 안정됩니다. 연결하자마자 값이 이상해도 당황하지 마세요.
:::

#### 코드 작성 및 실행 (15-25분)

**[실습 안내 스크립트]**

선생님: "Thonny 열고, 새 파일 만들어서 이 코드를 입력해보세요."

**[코드: step1_mq2_basic.py]**
```python
# === MQ2 가스 센서 기본 읽기 ===
# 센서의 아날로그 값을 읽어서 화면에 출력하는 코드

from machine import ADC, Pin
import time

# --- 센서 설정 ---
# Grove A0 포트 = GP26 (Pico의 ADC 채널 0)
gas_sensor = ADC(Pin(26))

# --- 반복해서 값 읽기 ---
while True:
    value = gas_sensor.read_u16()  # 0~65535 사이 값 읽기
    print(value)                   # 값 출력
    time.sleep(1)                  # 1초 대기
```

선생님: "실행해보세요. 숫자가 쭉 나오죠? 그게 지금 이 교실 공기의 가스 농도예요."

**[예상 Q&A]**

:::tip[값이 계속 변하는데 고장인가요?]
아니요, 정상이에요! 센서는 물리적 장치라서 아주 미세하게 값이 흔들려요. 이걸 '노이즈'라고 합니다. 대략적인 범위가 비슷하면 정상이에요.
:::

:::tip[read_u16이 뭐예요?]
read unsigned 16-bit의 줄임말이에요. 부호 없는(unsigned) 16비트 정수로 읽겠다는 뜻이에요. 그래서 0~65535 범위가 되는 거예요.
:::

:::tip[값이 엄청 높게 나와요 / 0에 가깝게 나와요]
MQ2는 예열이 필요합니다. 1-2분만 기다려보세요. 그래도 이상하면 Grove 케이블이 A0 포트에 제대로 꽂혀있는지 확인해보세요.
:::

:::tip[ADC(Pin(26))에서 26은 뭐예요?]
Pico의 GP26 핀이에요. Grove Shield에서 A0 포트가 GP26에 연결돼 있거든요. A1은 GP27, A2는 GP28이에요.
:::

**[코드: step2_mq2_plotter.py]**
```python
# === MQ2 가스 센서 + 플로터 시각화 ===
# 센서 값을 읽고, 노이즈를 줄여서 안정적으로 표시하는 코드

from machine import ADC, Pin
import time

# --- 센서 설정 ---
gas_sensor = ADC(Pin(26))

# --- 이동 평균 필터 함수 ---
# 왜 필요한가? 센서 노이즈를 줄여서 진짜 변화만 보려고
def read_average(sensor, count=10):
    """count번 읽어서 평균을 구하는 함수"""
    total = 0
    for i in range(count):
        total += sensor.read_u16()
        time.sleep_ms(10)  # 10ms 간격으로 읽기
    return total // count  # 정수 나눗셈

# --- 메인 루프 ---
while True:
    # 10번 읽어서 평균 → 노이즈 줄어듦
    avg_value = read_average(gas_sensor, 10)

    # Thonny 플로터용 출력
    print(avg_value)

    time.sleep(0.5)  # 0.5초마다 측정
```

**[예상 Q&A]**

:::tip[이동 평균이 뭐예요?]
쉽게 말하면 '여러 번 읽어서 평균 내기'예요. 시험 한 번 보면 운이 작용하지만, 열 번 보면 실력이 나오잖아요? 같은 원리예요.
:::

:::tip[10번 말고 100번 읽으면 더 정확해요?]
더 부드러워지긴 하는데, 그만큼 읽는 데 시간이 오래 걸려요. 실시간 반응이 느려지죠. 10번은 속도와 정확도의 균형점이에요.
:::

:::tip[time.sleep_ms(10)이랑 time.sleep(0.01)이랑 같은 건가요?]
네, 거의 같아요! sleep_ms는 밀리초 단위로 더 정밀하게 지정할 수 있어서 센서 읽기에 자주 써요.
:::

:::tip[손소독제 가까이 대니까 값이 엄청 올라가요!]
MQ2가 알코올 성분에 특히 민감하게 반응하거든요. 실제로 음주측정기에도 비슷한 원리의 센서가 들어있어요.
:::
