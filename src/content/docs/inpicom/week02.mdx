**[코드: step1_all_color.py]**
```python
# === RGB LED Stick: 전체 색상 제어 ===
# LED 10개를 한 번에 같은 색으로 켜는 코드

from machine import Pin
from neopixel import NeoPixel
import time

# --- LED 설정 ---
NUM_LEDS = 10
np = NeoPixel(Pin(16), NUM_LEDS)

# --- 색상 사전 (자주 쓰는 색) ---
RED    = (255, 0, 0)
GREEN  = (0, 255, 0)
BLUE   = (0, 0, 255)
YELLOW = (255, 255, 0)
PURPLE = (128, 0, 128)
CYAN   = (0, 255, 255)
WHITE  = (255, 255, 255)
OFF    = (0, 0, 0)

# --- 전체 LED를 한 색으로 설정하는 함수 ---
def fill(color):
    for i in range(NUM_LEDS):
        np[i] = color
    np.write()

# --- 색상 퍼레이드 ---
colors = [RED, GREEN, BLUE, YELLOW, PURPLE, CYAN, WHITE]
names  = ["빨강", "초록", "파랑", "노랑", "보라", "청록", "흰색"]

for color, name in zip(colors, names):
    fill(color)
    print(f"{name} 점등!")
    time.sleep(1)

fill(OFF)
print("끝!")
```

```python
# === 개별 LED 제어 + 색상 실험 ===
# LED 하나하나를 다른 색으로 켜는 코드

from machine import Pin
from neopixel import NeoPixel
import time

NUM_LEDS = 10
np = NeoPixel(Pin(16), NUM_LEDS)

# --- 1) 하나씩 켜기: 순서대로 파란 불이 이동 ---
for i in range(NUM_LEDS):
    # 먼저 전체 끄기
    for j in range(NUM_LEDS):
        np[j] = (0, 0, 0)
    # i번 LED만 켜기
    np[i] = (0, 0, 255)
    np.write()
    time.sleep(0.2)

# --- 2) 10개를 각각 다른 색으로 ---
# 빨강에서 보라까지 그라데이션
for i in range(NUM_LEDS):
    r = 255 - (i * 25)       # 빨강: 255→30 (점점 감소)
    b = i * 25               # 파랑: 0→225 (점점 증가)
    np[i] = (r, 0, b)
np.write()
time.sleep(3)

# --- 3) 도전! 자기만의 색 만들기 ---
# 아래 값을 바꿔보세요 (0~255)
my_r = 255   # ← 바꿔보세요!
my_g = 100   # ← 바꿔보세요!
my_b = 50    # ← 바꿔보세요!

for i in range(NUM_LEDS):
    np[i] = (my_r, my_g, my_b)
np.write()
print(f"내 색: ({my_r}, {my_g}, {my_b})")
```

**[예상 Q&A]**

:::tip[LED 순서가 거꾸로인데요?]
LED Stick의 IN/OUT 방향에 따라 달라요. 거꾸로면 케이블을 반대로 꽂았을 가능성이 있어요. 뒤집어 꽂아보세요.
:::

:::tip[(255, 255, 255)는 무슨 색이에요?]
흰색이에요! 빛의 3원색을 다 섞으면 흰색이 되거든요. 물감이랑 반대죠? 물감은 다 섞으면 검정, 빛은 다 섞으면 흰색.
:::

:::tip[주황색은 어떻게 만들어요?]
(255, 165, 0)이면 주황색이에요. 빨강을 최대로, 초록을 중간, 파랑을 0으로 하면 돼요. 인터넷에 'RGB color picker'를 검색하면 원하는 색의 값을 찾을 수 있어요.
:::

:::tip[밝기를 줄이고 싶어요]
모든 값을 같은 비율로 줄이면 돼요. (255,0,0)의 절반 밝기는 (128,0,0). 비율은 유지하면서 숫자만 줄이세요.
:::

#### 무지개 흐르기 (20-30분)

**[실습 안내 스크립트]**

선생님: "이제 진짜 재미있는 거 해봅시다. 무지개가 LED 위를 흘러가는 애니메이션이에요."

선생님: "무지개를 만들려면 RGB 값을 부드럽게 바꿔야 해요. 이걸 쉽게 해주는 함수를 알려줄게요."

**[시각 자료]**

```
HSV에서 무지개 만드는 원리 (wheel 함수):

  위치(pos)  0          85         170        255
            |────────────|────────────|──────────|
  색상     빨강  →  초록  →  파랑  →  빨강

  0~85:    R 감소, G 증가, B = 0
  85~170:  R = 0, G 감소, B 증가
  170~255: R 증가, G = 0, B 감소

  → 0~255 숫자 하나로 무지개 전체 색상을 표현!
```

**[코드: step3_rainbow.py]**
```python
# === 무지개 흐르기 애니메이션 ===
# 10개 LED 위를 무지개가 흘러가는 효과

from machine import Pin
from neopixel import NeoPixel
import time

NUM_LEDS = 10
np = NeoPixel(Pin(16), NUM_LEDS)

def wheel(pos):
    """0~255 값을 무지개 색상(R,G,B)으로 변환하는 함수
    0=빨강 → 85=초록 → 170=파랑 → 255=빨강"""
    if pos < 85:
        return (255 - pos * 3, pos * 3, 0)
    elif pos < 170:
        pos -= 85
        return (0, 255 - pos * 3, pos * 3)
    else:
        pos -= 170
        return (pos * 3, 0, 255 - pos * 3)

def rainbow_flow(wait=0.05):
    """무지개가 LED 위를 흘러가는 애니메이션"""
    for offset in range(256):
        for i in range(NUM_LEDS):
            # 각 LED에 위치별로 다른 색 배치
            color_pos = (i * 256 // NUM_LEDS + offset) % 256
            np[i] = wheel(color_pos)
        np.write()
        time.sleep(wait)

# --- 무지개 3바퀴 ---
print("무지개 시작!")
for _ in range(3):
    rainbow_flow(0.03)

# 끄기
for i in range(NUM_LEDS):
    np[i] = (0, 0, 0)
np.write()
print("끝!")
```

선생님: "실행하면 무지개가 흘러갈 거예요. 핵심은 wheel() 함수예요. 0에서 255 사이 숫자를 넣으면 무지개 위의 해당 색이 나와요."

선생님: "wait 값을 바꾸면 속도가 달라져요. 0.01로 하면 빠르게, 0.1로 하면 느리게."

**[예상 Q&A]**

:::tip[wheel 함수가 너무 복잡해요]
외울 필요 없어요! 원리만 이해하세요. 0~255 숫자를 넣으면 무지개 색이 나오는 '마법 함수'라고 생각하면 됩니다. 실제로 거의 모든 NeoPixel 프로젝트에서 이 함수를 복붙해서 써요.
:::

:::tip[% 256이 뭐예요?]
나머지 연산이에요. 255 다음에 다시 0으로 돌아가게 해줘요. 그래야 색이 끊기지 않고 계속 순환하거든요.
:::

:::tip[속도를 더 빠르게 하고 싶어요]
wait=0.01로 바꿔보세요. 더 빠르게 하면 time.sleep(0.005)도 가능하지만, 너무 빠르면 눈이 인식을 못 해요.
:::

**[코드: step4_my_pattern.py]**
```python
# === 나만의 LED 패턴 만들기 ===
# 아래 패턴 중 원하는 것을 골라 수정해보세요!

from machine import Pin
from neopixel import NeoPixel
import time

NUM_LEDS = 10
np = NeoPixel(Pin(16), NUM_LEDS)

# ── 패턴 A: 숨쉬기 (한 색이 천천히 밝아졌다 어두워짐) ──
def breathe(r, g, b, speed=0.03):
    """숨쉬기 효과 - r,g,b에 원하는 색 비율 입력"""
    # 밝아지기
    for brightness in range(0, 100, 2):
        scale = brightness / 100
        for i in range(NUM_LEDS):
            np[i] = (int(r*scale), int(g*scale), int(b*scale))
        np.write()
        time.sleep(speed)
    # 어두워지기
    for brightness in range(100, 0, -2):
        scale = brightness / 100
        for i in range(NUM_LEDS):
            np[i] = (int(r*scale), int(g*scale), int(b*scale))
        np.write()
        time.sleep(speed)

# ── 패턴 B: 나이트라이더 (불빛이 왔다갔다) ──
def knight_rider(r, g, b, speed=0.08):
    """좌우로 왕복하는 효과"""
    # 오른쪽으로
    for i in range(NUM_LEDS):
        for j in range(NUM_LEDS):
            np[j] = (0, 0, 0)
        np[i] = (r, g, b)
        # 잔상 효과: 양쪽에 어두운 빛
        if i > 0:
            np[i-1] = (r//4, g//4, b//4)
        if i < NUM_LEDS - 1:
            np[i+1] = (r//4, g//4, b//4)
        np.write()
        time.sleep(speed)
    # 왼쪽으로
    for i in range(NUM_LEDS - 2, 0, -1):
        for j in range(NUM_LEDS):
            np[j] = (0, 0, 0)
        np[i] = (r, g, b)
        if i > 0:
            np[i-1] = (r//4, g//4, b//4)
        if i < NUM_LEDS - 1:
            np[i+1] = (r//4, g//4, b//4)
        np.write()
        time.sleep(speed)

# ── 패턴 C: 깜빡이 경광등 (빨강/파랑 교대) ──
def police_light(speed=0.15):
    """경광등 효과"""
    for _ in range(5):
        # 왼쪽 반 빨강, 오른쪽 반 파랑
        for i in range(5):
            np[i] = (255, 0, 0)
        for i in range(5, 10):
            np[i] = (0, 0, 255)
        np.write()
        time.sleep(speed)
        # 반전
        for i in range(5):
            np[i] = (0, 0, 255)
        for i in range(5, 10):
            np[i] = (255, 0, 0)
        np.write()
        time.sleep(speed)

# =============================================
#  아래에서 원하는 패턴을 실행하세요!
#  색상과 속도를 자유롭게 바꿔보세요.
# =============================================

# 패턴 A: 숨쉬기 (색상 바꿔보세요!)
for _ in range(3):
    breathe(0, 100, 255)   # 하늘색 숨쉬기

# 패턴 B: 나이트라이더 (색상 바꿔보세요!)
for _ in range(3):
    knight_rider(255, 0, 0)  # 빨간 나이트라이더

# 패턴 C: 경광등
police_light(0.1)

# 끄기
for i in range(NUM_LEDS):
    np[i] = (0, 0, 0)
np.write()
```

```
└──────────────────────────────────────────┘
```

**[예상 Q&A]**

:::tip[int()는 왜 씌우는 거예요?]
scale을 곱하면 12.5 같은 소수가 나올 수 있어요. 근데 RGB 값은 정수만 됩니다. int()로 소수점을 버리는 거예요.
:::

:::tip[r//4는 뭐예요?]
정수 나눗셈이에요. 255//4 = 63. 밝기를 1/4로 줄여서 '잔상' 효과를 만드는 거예요.
:::

:::tip[패턴을 섞고 싶어요]
그냥 순서대로 호출하면 돼요! breathe 다음에 knight_rider 호출하면 자연스럽게 이어져요.
:::

:::tip[새 패턴을 만들고 싶은데 어디서부터요?]
기존 패턴을 복사해서 이름을 바꾸고, for문 안의 색상 계산만 수정하세요. 나이트라이더에서 이동 방향을 바꾸거나, 숨쉬기에서 LED마다 시간차를 주거나.
:::
