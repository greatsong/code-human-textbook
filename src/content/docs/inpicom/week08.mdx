3. 3개 부품을 통합한 시스템을 설계하고 구현할 수 있다

:::note[상태 머신이란?]
상태 머신(State Machine)은 시스템이 가질 수 있는 **상태**와 상태 간의 **전환 조건**을 정의하는 설계 방법입니다. 자판기, 신호등, 게임 캐릭터 등 일상의 많은 시스템이 상태 머신으로 설계됩니다.
:::

#### 상태 전환 기본 코드 (12-20분)

**[코드: step1_state_machine.py]**
```python
# === 상태 머신 기본 구조 ===
# 세 가지 상태 전환 패턴 학습

from machine import Pin, ADC
from neopixel import NeoPixel
import time

# --- 하드웨어 ---
pir = Pin(18, Pin.IN)
gas = ADC(Pin(26))
NUM_LEDS = 10
np = NeoPixel(Pin(16), NUM_LEDS)

# --- 상태 정의 ---
STATE_SLEEP  = "SLEEP"
STATE_ACTIVE = "ACTIVE"
STATE_ALERT  = "ALERT"

current_state = STATE_SLEEP

# --- LED 표시 함수 ---
def show_sleep():
    """절전: LED 1개만 파란색 (나머지 끔)"""
    for i in range(NUM_LEDS):
        np[i] = (0, 0, 0)
    np[0] = (0, 0, 20)  # 아주 약한 파랑
    np.write()

def show_active():
    """활성: 초록색"""
    for i in range(NUM_LEDS):
        np[i] = (0, 50, 0)
    np.write()

def show_alert():
    """경보: 빨강"""
    for i in range(NUM_LEDS):
        np[i] = (255, 0, 0)
    np.write()

# --- 메인 루프 ---
print("=== 상태 머신 테스트 ===")
show_sleep()

while True:
    if current_state == STATE_SLEEP:
        if pir.value() == 1:
            print("SLEEP → ACTIVE (움직임 감지)")
            current_state = STATE_ACTIVE
            show_active()

    elif current_state == STATE_ACTIVE:
        value = gas.read_u16()
        if value > 30000:  # 가스 감지
            print(f"ACTIVE → ALERT (가스: {value})")
            current_state = STATE_ALERT
            show_alert()

    elif current_state == STATE_ALERT:
        value = gas.read_u16()
        if value < 20000:  # 가스 해소
            print("ALERT → ACTIVE (가스 해소)")
            current_state = STATE_ACTIVE
            show_active()

    time.sleep(0.5)
```

### 실습 2: 절전형 경보기 (20-40분)

#### 완성 코드 (20-35분)

**[코드: step2_power_save.py]**
```python
# === 절전형 가스 경보기 ===
# PIR 감지 시에만 MQ2 측정, 타임아웃 시 절전 복귀

from machine import Pin, ADC
from neopixel import NeoPixel
import time

# --- 하드웨어 ---
pir = Pin(18, Pin.IN)
gas = ADC(Pin(26))
NUM_LEDS = 10
np = NeoPixel(Pin(16), NUM_LEDS)

# --- 설정 ---
GAS_WARN = 20000
GAS_ALERT = 35000
TIMEOUT_MS = 10000   # 10초 무감지 시 절전

# --- 상태 ---
STATE_SLEEP  = 0
STATE_ACTIVE = 1
STATE_ALERT  = 2
state = STATE_SLEEP
last_motion = 0

# --- 이동 평균 ---
def read_avg(sensor, n=10):
    total = 0
    for _ in range(n):
        total += sensor.read_u16()
        time.sleep_ms(10)
    return total // n

# --- LED 함수들 ---
def led_sleep():
    for i in range(NUM_LEDS):
        np[i] = (0, 0, 0)
    np[0] = (0, 0, 15)  # 상태 표시: 약한 파랑 1개
    np.write()

def led_active(gas_val):
    """가스 농도에 따라 초록~노랑 그라데이션"""
    ratio = min(gas_val / GAS_ALERT, 1.0)
    r = int(255 * ratio)
    g = int(255 * (1 - ratio * 0.5))
    for i in range(NUM_LEDS):
        np[i] = (r, g, 0)
    np.write()

def led_alert_blink():
    for i in range(NUM_LEDS):
        np[i] = (255, 0, 0)
    np.write()
    time.sleep(0.15)
    for i in range(NUM_LEDS):
        np[i] = (0, 0, 0)
    np.write()
    time.sleep(0.15)

# --- 메인 루프 ---
print("=== 절전형 가스 경보기 ===")
led_sleep()

while True:
    motion = pir.value()
    now = time.ticks_ms()

    if motion:
        last_motion = now

    # === 상태별 처리 ===
    if state == STATE_SLEEP:
        if motion:
            state = STATE_ACTIVE
            print("💤→👁️ 활성화!")
        else:
            time.sleep(0.5)  # 절전: 느리게 체크
            continue

    if state == STATE_ACTIVE:
        gas_val = read_avg(gas)
        led_active(gas_val)

        if gas_val >= GAS_ALERT:
            state = STATE_ALERT
            print(f"👁️→🚨 경보! (가스: {gas_val})")

        # 타임아웃: 일정 시간 움직임 없으면 절전
        elapsed = time.ticks_diff(now, last_motion)
        if elapsed > TIMEOUT_MS:
            state = STATE_SLEEP
            led_sleep()
            print("👁️→💤 절전 복귀")

    elif state == STATE_ALERT:
        led_alert_blink()
        gas_val = read_avg(gas)

        if gas_val < GAS_WARN:
            state = STATE_ACTIVE
            print(f"🚨→👁️ 경보 해제 (가스: {gas_val})")

    print(f"[{['SLEEP','ACTIVE','ALERT'][state]}] motion={motion}")
    time.sleep(0.3)
```

**[예상 Q&A]**

Q: "continue가 뭐예요?"
A: "while 루프의 나머지를 건너뛰고 처음으로 돌아가라는 뜻이에요. 절전 모드에서는 가스 측정을 안 해도 되니까, continue로 바로 다음 루프로 가는 거예요."

Q: "실제로 배터리가 오래 가나요?"
A: "Pico 자체를 deep sleep 모드로 넣으면 더 절약되지만, 오늘은 소프트웨어 수준의 절전이에요. 그래도 LED를 끄고 센서를 안 읽는 것만으로도 상당한 차이가 나요."

:::tip[continue 키워드]
`continue`는 while 루프에서 **나머지 코드를 건너뛰고** 루프의 처음으로 돌아갑니다. 절전 모드에서는 가스 측정이 불필요하므로, `continue`로 바로 다음 PIR 체크로 돌아가는 것이 효율적입니다.
:::

### 정리 (40-50분)

```
상태 머신 패턴 정리:

1. 상태를 정의한다 (SLEEP, ACTIVE, ALERT)
2. 각 상태에서의 동작을 정한다 (LED 표시, 센서 읽기)
3. 상태 전환 조건을 정한다 (PIR 감지, 가스 임계값, 타임아웃)

이 패턴은 어디서나 쓰여요:
• 자판기: 대기 → 투입 → 선택 → 배출
• 신호등: 초록 → 노랑 → 빨강
• 게임: 대기 → 진행 → 승리/패배
```
